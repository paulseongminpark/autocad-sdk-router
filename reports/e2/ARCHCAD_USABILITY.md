# ArchCAD 사용 가능성 판정 (실측 기반)

작성 2026-07-20 (effort max). 질문: "받은 ArchCAD 40k를 쓸 수 있나?" 답: **용도에 따라 다르다 —
이진이 아니다.** 아래는 디스크 실측으로 확정한 사실과 용도별 판정.

## 실측 사실 (payload 검사, 발명 없음)
- 위치: `D:\datasets\ArchCAD_hf_full\data\{png,json,svg,point,caption}.zip`
- 규모: **41,097 샘플 × 5 모달**, UUID로 완전 정렬(svg∩json = 41,097 전건 일치).
- 파일명: **랜덤 UUID**(예: `3e6157f6-28d6-44a9-97d3-ed56570821b1`) — `<도면>-<슬라이스>` 패턴
  아님. 도면 그룹핑 키 없음.
- JSON payload: top-level `{entities:[...]}`, **출처/도면/건물/프로젝트 필드 0개**.
- Caption payload: `[{question,answer}...]`, **출처 필드 0개**.
- 라벨: semantic(벽=ID 20) + instance 존재(README + entities semantic 필드).

## 결정적 함의 — 누수 안전성
각 샘플은 평면도의 14m×14m **슬라이스**다. 전체 코퍼스에서 5,538 도면 → 413,062 청크
(≈75 슬라이스/도면). 즉 **한 건물이 여러 슬라이스를 낳는다.** 그런데 40k 공개본은 슬라이스마다
랜덤 UUID를 부여하고 **어느 건물에서 왔는지를 삭제**했다(파일명·payload 어디에도 없음). 따라서
우리가 train/test를 어떻게 나눠도 **같은 건물의 슬라이스가 양쪽에 섞일 수 있다 = 누수(leakage)**.
이 데이터만으로는 복구 불가능한 문제다.

## 용도별 판정
| 용도 | 가능? | 근거 |
|---|---|---|
| ArchCAD **내부** 일반화 벤치마크 (건물A 학습→건물B 시험) | ❌ 불가 | 도면-격리 분할 불가(그룹핑 키 삭제). 누수 못 막음 |
| **보조 학습원** (사전학습·표현학습·증강) | ✅ 가능 | 41k 벽-라벨 슬라이스. 단 **평가는 딴 데서** |
| **전이 실험 재료** (ArchCAD 학습 → CubiCasa/FloorPlanCAD 시험) | ✅ 가능·가치 높음 | 시험셋이 분할 완비된 딴 코퍼스면 누수 없음 |
| **규칙 귀납 소스** (D8 온톨로지 귀납) | ✅ 가능 | 라벨된 프리미티브 다량, held-out 주장 안 하면 OK |

**핵심 원리**: 누수는 "학습 데이터"가 아니라 **"평가(시험)를 어디서 하느냐"**에서 결정된다.
ArchCAD로 학습·증강·귀납하고, **시험은 분할이 온전한 CubiCasa5K(우리 val-A, COMPLETE) 또는
FloorPlanCAD 정본(방금 확보, 프로젝트-격리 분할)에서** 하면 전건 누수 안전.

## 내부 벤치마크 잠금 해제 조건
전체 400K 코퍼스 + 공식 도면-격리 7:1:2 분할 확보 시에만. 공개 경로 UNKNOWN → 저자 요청(B,
초안 `ARCHCAD_AUTHOR_REQUEST_DRAFT.md`)이 유일 경로.

## W4 편성 함의
- ArchCAD 40k = **전이/보조/귀납 트랙에 즉시 투입 가능** (M-11 청주가 둘째 프로젝트로 자격
  통과했으므로, ArchCAD는 셋째 라벨 소스 — 교차-데이터셋 전이 축 강화).
- ArchCAD 내부 벤치마크는 전체본 확보까지 보류(NON_GOAL 유지).
- eligible_for_training: **보조/전이 용도 = 허용**, 내부 held-out 주장 = 금지.
