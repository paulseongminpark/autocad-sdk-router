### 반증 (수용 — 4계열)

1. **가드 우회 4종 (lens1 B1~B4, 라이브)**: 가드 술어(ratio 보유 ∧ handle 미충돌)의 여집합이
   전부 열려 있다 — display 제거(B2)·handle 재사용(B3)·GRID 위장(B1)은 봉인 반례의 최소 변형으로
   C1v4에서 status LOW→HIGH를 재현하고, ratio-일치 위치 위조(B4)는 봉인 필드 3종 전부를 올린다.
   300종 속성 시험의 생성 문법이 이 경로들을 표본화하지 않았음(사각 일치)도 규명됨.
2. **가드 유발 신종 역전이 — 분모 청소 (seat4 §5.1, 라이브)**: stale 교란이 reference-outlier를
   ratio-outlier로 만들면 v2가 그것을 합의 분모에서 제거해 consensus 4/5→4/4, 신뢰도 0.64→0.80으로
   HIGH 돌파. 원본은 LOW 유지. 54종 전개 중 15종 재현. **수리가 새 결함을 도입했다.**
3. **정당 입력 과차단 (lens1 O1~O3 · seat4 §5.3)**: 무교란 이중 치수 병기(1000MM≡1M) 장면에서
   원본 대비 status HIGH→LOW의 coverage 손실 — 도시에 §2.3의 z^mm(단위 정규화) 합의 규칙을 구현이
   따르지 않는 것이 근인. mode 탈취 시 진짜 앵커 격리·위조 전용 지지(O3)도 실측.
4. **이중 봉인 선행 입증 실패 + 검사기 결함 (seat4 §1.2)**: 수리-전 워크북의 실물 부재, 전달
   워크북 시각이 수리 코드보다 늦음, 검사기가 SHA 불일치 시 시간 검사를 건너뛰고
   `created_before_estimator=True`를 상수 반환. prereg.json의 시간 선행만 성립(17:49 < 17:55).

### 오케스트레이터 자기 귀책 판정 — 봉인 문언의 정보이론적 과잉

lens1 B4는 양 추정기 공통 상승이다: 진짜 지지 증거와 분포상 구별 불가능한 위조 레코드는 어떤
지지-계수 추정기의 신뢰도도 올린다. 내 L1c 패킷 문언("어떤 교란도 올릴 수 없음")을 워커가 그대로
`scope: any perturbation`으로 봉인했고, 이 밴드는 **충족 불가능**하다. 도시에의 실제 계약은 지정
mutation family 스코프다. 과잉 문언은 내 귀책. L1d의 봉인은 도시에-유도 밴드로 한다: 지정 6
family + 이번 함대가 발견한 신규 class 전부(B1·B2·B3·분모청소·혼합단위 무손실) — 이는 밴드 완화가
아니라 SoT(도시에) 준거로의 교정이며, B4(완전 위조)는 게이트가 아닌 측정·문서화 항목으로 남긴다.

## L1d 수리 계약의 구조 원칙 (함대 발견의 직접 번역)

1. **입장 가드 폐기, 합의 계산 재설계**: ① ratio 공간은 mode 선택 **전에** z^mm 단위 정규화
   (도시에 §2.3:205 — O1/§5.3 해소) ② 독립성 n_independent = **고유 source handle 수**
   (도시에 §2.2.4 — B3 해소) ③ span 증거는 label-free 유지, ratio 정합을 span 채택에 결합 금지
   (도시에 §2.4 — O2 해소).
2. **단조성의 구조적 달성 — "의심 증거는 분모에만"**: 교차-공간 정합에 실패한 레코드는 모든 신뢰도
   인자의 분모(의심)에는 남고 분자(지지)에는 못 든다. 레코드 제거·무시가 신뢰도를 올리는 경로
   자체를 제거(분모 청소 해소; B1·B2류는 지지 불가로 무력화).
3. **검사기 무결**: 봉인 선행은 실물로 — 수리-전 워크북 사본을 불변 sidecar로 영구 보존, 해시는
   실제 바이트에서 계산, 상수 반환 금지.
4. **속성 시험 문법 확장**: 함대 프로브 전 계열(lens1 B1~B4·O1~O3, seat4 54-sweep·혼합단위·
   handle 충돌)을 회귀로 영구 편입 + 교란 문법에 display 제거·handle 충돌·type 변경 추가.
