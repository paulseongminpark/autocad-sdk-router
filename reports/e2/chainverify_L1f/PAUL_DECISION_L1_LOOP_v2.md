# PAUL 결재 요청 v2 — L1 루프 (듀오 검토 반영, v1의 A/B/C/D 프레임 폐기)

v1(같은 폴더)은 오케스트레이터 단독 작성이었고, 듀오 검토(sol ultra + Fable 독립 좌석,
`D:\runs\e2_program\duo_decision_review\{sol,fable}_position.md`)가 이를 기각·재설계했다.
합치점: ① v1의 권고 A는 문구 그대로 구현 불가(표면 바이트-재생산 = 항진 딜레마, 코드 증명)
② "수리 1~2회" 비용 주장 근거 없음(통과율 0/6 → 정직 추정 = 함대 라운드 2±1회) ③ B·D 기각
유지 ④ **C는 배타 선택지가 아니라 즉시 안전조치** — 4지선다 프레임 자체가 오류.

## 개정 권고 — 단일안 "지금 격리 + 봉인 게이트 뒤 재설계(A′)"

1. **즉시 (오늘)**: L1 추정기를 자기인증 계측기에서 **참고 신호로 강등**. 증인 서사 기계 삭제
   (6판간 적발 0건·90/90 복사·템플릿 2개로 판명). C2는 원하면 낮은 천장으로 가역 개방 가능
   (A′ 착지 시 재자격·천장 복원).
2. **밴드 재입법**: "위조 인증 0"의 개방-우주 보편 주장 철회(6판 비수렴의 구조적 원인) →
   ⑴ 기계 판정 가능한 구성 조건 + ⑵ 봉인 누적 공격 코퍼스(1~5차 함대 전 프로브) 및 신규 적대
   좌석에 대한 위반 0, 으로 이원화. 성문 else-분기(회귀-only 기각 처리) 완결 포함.
3. **A′ 재설계 — 사전 봉인된 타당성 게이트(A0) 뒤에서만**:
   - 파생-재생 검증기: 자유 필드는 봉인 스키마(화이트리스트·타입·정의역)로 **입구에서 파서-수준
     거부**(형 공백 1,024건 class 구조적 소거), 파생 필드는 전량 규칙 재유도 후 바이트 대조.
   - 물리 동일성 정규화: 동일 (p0,p1)·공유 source_span_id의 상이-핸들 = 단사 공리 위반 경성
     차단(독립성 위조 class 소거 — 정직 실측 0/400·0/1028 근거).
   - 공통 수리 동반: grid_present 중립성 위반·속성 하니스의 실코퍼스 표집.
   - 게이트 술어는 구현 자기보고가 아니라 **독립 판정기**가 실행.
   - 다섯 함대 누적 공격 코퍼스 통과 시에만 재자격 심사. 실패/비용 초과 시 강등 상태 유지.
4. **비용 정직화**: 함대 라운드 2±1회. 자격 주장은 봉인 합성 세계 한정 — 실도면(C2 포함) 투입
   전 실코퍼스 캘리브레이션 게이트 별도.

**결재 문항 (한 줄)**: 위 단일안(즉시 강등 + A0 게이트 뒤 A′ 재설계)을 승인하는가,
아니면 C-종결(재설계 없이 강등으로 마감)로 가는가.
