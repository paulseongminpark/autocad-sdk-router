# PAUL 결재 요청 — L1 수리 루프의 다음 단계 (2026-07-19)

## 한 문단 요약 (여기만 읽어도 결정 가능)

E2 프로그램에는 "축척 추정기"라는 부품이 있습니다 — 도면 속 치수 표기(앵커)들을 모아 그 도면의
단위 배율을 추정하고, 그 추정을 얼마나 믿어도 되는지(신뢰도)를 스스로 매기는 계측기입니다.
이 신뢰도가 절대 하면 안 되는 일이 하나 있습니다: **조작된 장면을 보고 신뢰도가 올라가는 것**
(위조 인증). 우리는 "위조 인증 0건" 규칙을 사전 봉인하고, 이를 만족하는 추정기를 만드는 수리
루프(L1)를 돌려 왔습니다. 매 수리마다 독립 검증단 4석(Claude Fable 3석 시야분리 + OpenAI sol
1석)이 깨뜨리기를 시도했고, **5차례 연속 기각**됐습니다. 다섯 번째 기각에서 사전에 성문화해 둔
중단 조건("새로운 종류의 결함으로 기각되면 수리 반복을 멈추고 Paul에게 설계 결정을 올린다")이
성립했습니다. 수리를 거듭할수록 좁아지는 게 아니라, 검사하지 않은 축에서 새 구멍이 계속
나오는 패턴이 실증됐기 때문입니다. **아래 선택지 A/B/C/D 중 하나의 결정이 필요합니다.**

## 루프 전 이력 (요구 조항: 전 이력 첨부)

| 회차 | 수리 내용 | 검증단 평결 | 적중한 결함 |
|---|---|---|---|
| L1 | 1차 수리 | 기각 | 특정 장면 가족을 조용히 희생시켜 지표를 맞춤 (모집단 마스킹) |
| L1b | 게이트 수치 충족 | 기각 (2C/2R) | "수리됐다"던 결함이 사실 새 모집단에 가려 잠복해 있었음 + 재현 주장 과장 |
| L1c | 입장 가드 추가 | 기각 (2C/2R) | 가드 우회 4종 + 가드 자체가 만든 새 결함 (분모 청소) |
| L1d | 합의 로직 재설계 | 기각 (2C/2R) | 신뢰도가 0 아니면 1로 몰리는 절벽 분포 → 시험 대부분이 만점에서 시작해 판별력 상실 (포화-은폐) |
| L1e | 연속 감쇠 + 증인 원칙 + git 봉인증인 | 기각 (1C/3R) | 의심 탐지기 맹점 3종 + 희석 공격(깨끗한 증거를 잔뜩 넣으면 평균이 의심 신호를 씻어냄) + 증인 항진 |
| L1f | 2계층 의심 분류 + 희석-불가 하한 | **기각 (1C/3R) ← 이번** | 아래 상세 |

- 검증단 산출물: `reports/e2/chainverify_L1b|L1c|L1d|L1e|L1f/` (각 4석 평결 + 종합 판정, 전부 커밋).
- 봉인 계약: 각 수리마다 코드 작성 **전에** 통과 조건을 봉인(git 커밋으로 선행 증명 — 4·5차
  검증단이 커밋 내용물 수준으로 이 선행성을 실증).

## 이번(5차) 판정의 핵심

**진짜로 고쳐진 것** (검증단이 독립 재현으로 확증):
- 희석 공격 축 — 깨끗한 증거를 400개 넣어도 의심 신호의 벌점이 비트 단위로 불변 (공식 수준 증명).
- 위조 표면의 수치 축 — 잔존 상승 44건의 사후 표면에 부정 신호 잔존 0 (3중 교차 확증).
- 수치 장부 — 5차에 걸쳐 실행 보고의 어떤 숫자도 재구성에 실패한 적 없음 (수치 조작은 한 번도 없었음).

**이번에 뚫린 것**:
1. **[신규 class] 선언 필드의 "형(type)" 심판 공백** — 필드에 계약과 다른 자료형(숫자 자리에
   문자열 등)이 들어 있어도 심판 규칙이 못 보고, 그 상태로 신뢰도 상승 + 최고 등급 + 정직
   인증까지 통과 (반례 1,024/1,024).
2. **독립성 위조** — "독립적인 지지 증거 3개"를 셀 때 물리적으로 같은 측정의 복사본(같은 위치
   중복, 같은 출처 ID 공유)을 별개 증거로 계수 → 신뢰도 위조 상승 (정직 데이터에선 절대 안
   나오는 표면인데 차단 규칙이 없음).
3. **증인 항진 (4차 지적, 미수리)** — "이 상승은 정직한 장면에서도 나올 수 있다"는 증명(증인)이
   실제로는 조작된 장면 자신을 복사한 것 (90/90건). 증인 조건을 진지하게 읽으면 잔존 상승 44건
   전부가 계약 위반이 됨.
4. 그 외: 정직한 격자(GRID) 앵커를 추가하면 오히려 신뢰도가 붕괴하는 과차단(4차 결함의 변주),
   속성 시험이 실제 코퍼스를 표집하지 않는 마스킹(1차 결함의 재발).

**패턴 진단**: 수리 6판을 관통하는 메타패턴은 "지적된 지점의 점(点)수리 → 검사하지 않은 축으로
구멍 이동"입니다. 값 축(4차) → 다중도 축(5차) → 자료형 축(5차)으로 같은 규칙("선언된 모든
필드의 모든 측면을 심판하라")의 새 부분 공백이 매번 나왔습니다. 이 패턴이 수렴한다는 증거가
없습니다 — 그래서 성문화된 중단선이 발동됐습니다.

## 결정 요청 — 선택지

**(A) 증인 개념의 재설계 (구성적 증인) — 권고안**
"정직한 장면에서도 이 상승이 나올 수 있다"를 말로/복사로 주장하는 대신, **정직 생성기를 실제로
실행해 그 표면을 재생산해 보이는 것만** 증인으로 인정하도록 규칙 자체를 다시 설계합니다.
증인이 항진(자기 복사)이 될 길이 원천 차단되고, "형 심판 공백" 류도 생성기가 못 만드는 표면은
자동으로 위조 판정됩니다. 비용: 봉인 조문 재입법 + 수리 1~2회 분량의 구현 (다만 이번엔 점수리가
아니라 심판 구조의 교체). 기존 자산(희석-불가 공식, 경성 차단, 봉인-증인 절차)은 전부 승계.

**(B) 밴드 재입법 (증인 요건 완화)**
"위조 인증 0"을 수치 조건만으로 판정하도록 완화. 즉시 통과 가능(수치 축은 이미 견고)하지만,
4·5차 검증단이 실증한 위조 경로(독립성 위조·형 공백)가 "정직"으로 인증되는 것을 허용하게 됨
— 봉인 취지의 후퇴라 **비권고**.

**(C) L1 동결 + 스코프 강등**
추정기를 "신뢰도를 자기 인증하는 계측기"가 아니라 "참고용 하위 신호"로 강등하고 0-violation
요구를 철회. E2 본선(탐지기 사다리)은 이 부품 없이도 진행 가능하므로 가장 싼 길이지만, C2
(절대 치수 vs 상대 비례 대결)와 실도면 축척 신뢰 파이프라인이 이 부품의 자격에 걸려 있어
그 트랙들의 천장이 낮아집니다.

**(D) 수리 반복 계속 (L1g)**
성문화된 중단 규칙 위반이므로 **비권고**. (이 규칙은 정확히 이 상황 — 끝없는 두더지잡기 —
을 막으려고 사전에 봉인해 둔 것입니다.)

**권고: A.** 근거 — 이번 함대가 남긴 것이 정확히 A의 재료입니다: lens1이 만든 "정직 생성계
실측 봉투" 방법론과 "생성계가 못 만드는 표면 = 위조" 원칙은 구성적 증인의 절반을 이미
구현했습니다. 실패 패턴(선언적 심판의 부분 공백)을 구조적으로 우회하는 유일한 안이기도 합니다.

## 결정까지의 상태

- L1 수리 반복: **중단됨** (성문 규칙 이행).
- C2 대결: **폐쇄 유지** (L1 자격에 의존).
- E2 본선(탐지기 사다리·W2 웨이브)은 이 결정과 독립적으로 계속 진행 중 — GNN 채택(val-A
  0.9748)·검증기 재자격·RL 열거 진단 등은 영향 없음.
