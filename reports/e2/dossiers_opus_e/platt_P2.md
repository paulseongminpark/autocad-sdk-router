# E2 방법론 심층 도시에 — platt_P2

**제안**: P2 — Graph IR + GNN 벽-멤버 분류기 (그래프·DL, 문맥 Δ-판별)
**좌석**: platt_strong_inferencer (P2)
**작성 원칙**: 수치 인용은 패킷 다이제스트(2026-07-18 세션 도구 출력)에서만. 그 외 수치·기법·논문은 문헌 일반 지식이며, 확신이 낮은 인용은 `[요검증]` 표기. 웹 검색 미사용.

---

## 0. 이 도시에가 방어하는 한 문장

**P2는 "GNN이 벽을 잘 맞춘다"를 증명하려는 제안이 아니다. "동일 진리·동일 스플릿에서, 관계적 문맥(relational context)을 빼면 벽 판별이 얼마나 무너지는가"를 계량하는 계측기다.** 산출물의 1급 시민은 분류기 성능이 아니라 **Δ(델타) = F1(문맥 포함) − F1(문맥 없는 최강 기하 P1)** 라는 하나의 숫자와, 그 숫자가 이름/레이어 피처를 가렸을 때(mask-ablation) 어떻게 변하는가이다. 성능은 부산물, 판별이 본체다.

이 프레이밍을 앞세우는 이유: GBDT가 이미 **val F1 0.517 (P 0.860 / R 0.370 / AUC 0.9215)** 라는 강한 지역(local) 기저를 세웠기 때문이다(다이제스트). 여기서 "GNN을 얹으면 더 오른다"는 흔한 결론은 과학적으로 공허하다. 값 있는 질문은 **"올랐다면 그 상승분이 진짜 다중홉 문맥(multi-hop context)의 기여인가, 1홉 이웃-집계로도 나오는가, 아니면 레이어 이름을 학습한 지름길(shortcut)인가"** 이다. 이 세 가지를 분리해서 재는 것이 P2의 존재 이유다.

## 1. 이론적 근거·선행연구

### 1.1 방법론 계보 — CAD 프리미티브 그래프 위의 노드 분류

P2의 핵심 관측 언어는 **"CAD 도면을 엔티티(선분·호·폴리라인)를 노드로, 기하 관계(평행·정션·오버랩)를 엣지로 하는 그래프로 보고, 각 노드에 '벽-멤버 여부' 이진 라벨을 분류한다"** 이다. 이는 새로운 발상이 아니라 이미 출판된 패러다임이다:

- **GAT-CADNet** (Zheng et al., CVPR 2022 `[요검증: 정확 저자/연도]`) — CAD 프리미티브를 노드, 관계를 엣지로 놓고 **Graph Attention Network**로 panoptic symbol spotting을 수행. **P2와 가장 가까운 선행 시스템**이다. 벽은 "symbol"의 일종은 아니지만, 프리미티브-그래프-위-노드분류라는 뼈대가 동일.
- **FloorPlanCAD / PanCADNet** (Fan et al., CVPR 2021 `[요검증]`) — 벡터 CAD 위 panoptic symbol spotting 데이터셋·기저 모델을 제시. 우리가 보유한 FloorPlanCAD 자산의 출처.
- **CADTransformer** (Fan et al., CVPR 2022 `[요검증]`) — CAD 프리미티브를 토큰화해 transformer로 처리. 그래프-메시지패싱 대신 attention-전역 결합을 쓴 계열로, P2의 GAT 암과 사촌.
- **SymPoint / "Symbol as Points"** (Liu et al., 2023 `[요검증]`) — 프리미티브를 포인트 집합으로 다루는 대안 표현. P2의 노드-표현 설계 시 참고할 반대 표현.

이 선행연구들의 함의: **벡터 CAD의 의미 분류를 GNN/attention으로 푸는 것은 학계에서 검증된 계보**이며, P2는 exotic한 도박이 아니다. 다만 이들은 대부분 **symbol spotting(도어·윈도·가구 등 블록형 심볼)** 을 겨냥하고, **벽(wall)** 은 심볼이 아니라 "연장된 평행 이중선의 집합"이라는 점에서 P2는 이들과 과업이 다르다 — 이 차이가 §3에서 P2의 고유 난제를 낳는다.

### 1.2 GNN 기본기 — 왜 이 아키텍처들인가

- **GraphSAGE** (Hamilton, Ying, Leskovec, NeurIPS 2017 `[요검증: 연도]`) — **inductive**(귀납적) 표현학습 + 이웃 샘플링. **P2의 1순위 백본**. 이유: transductive(직관적) GCN과 달리 *본 적 없는 그래프에 그대로 적용* 가능 → CubiCasa로 훈련한 모델을 1.dwg 실도면에 무재훈련 전이하는 우리 시나리오의 필수 조건. 집계 함수(mean/max/pool)가 명시적이라 "문맥이 무엇을 하는가"를 해석하기 쉬움.
- **GAT / Graph Attention Network** (Veličković et al., ICLR 2018 `[요검증]`) — 이웃마다 attention 가중. **평행 엣지 vs 정션 엣지 중 무엇을 더 볼지**를 학습으로 정하게 함. P2의 2순위 암(표현력↑, 파라미터↑).
- **GCN** (Kipf & Welling, ICLR 2017 `[요검증]`) — 스펙트럴 동기의 1홉 컨볼루션. transductive 한계로 P2에서는 기저-대조군용으로만.
- **GIN** (Xu et al., ICLR 2019 `[요검증]`) — Weisfeiler-Lehman 급 표현력. 벽 판별이 "구조적 모티프(예: 이중선-정션-이중선 사슬)를 세는" 문제로 드러나면 필요.
- **MPNN 통합 프레임** (Gilmer et al., ICML 2017 `[요검증]`) — 메시지=엣지피처+이웃상태, 갱신=집계라는 통일된 관점. 우리의 엣지 피처(평행성·gap·overlap)가 곧 "메시지"에 대응.
- **R-GCN / 관계형 GNN** (Schlichtkrull et al., 2018 `[요검증]`) — 엣지-타입별 별도 가중행렬. **벽 그래프는 동종성(homophily)과 이종성(heterophily)이 섞여 있다** — 평행 엣지 상대는 대개 같은 벽(동종), 정션 엣지 상대는 도어·윈도(이종). 기본 GNN은 동종성을 가정하므로(Zhu et al. 2020 "Beyond Homophily" `[요검증]`), 엣지-타입 분리(R-GCN)가 이 혼합을 다루는 정공법. P2의 중요한 설계 축.

### 1.3 노이즈 라벨·PU 학습 — silver를 다루는 계보

- **Positive-Unlabeled (PU) learning** (Elkan & Noto 2008; du Plessis et al. nnPU 2015 `[요검증]`) — silver 라벨은 신뢰 못 할 양성(positive)이고, 미라벨은 "양성+음성 혼합"으로 봐야 함. 다이제스트의 silver 역할일치 **0.5491** 세계에서, silver를 hard label로 쓰면 노이즈를 학습한다. nnPU/대칭 CE로 완화.
- **Co-teaching / 노이즈-강건 손실** (Han et al. 2018 `[요검증]`) — 두 네트워크가 서로의 저손실 샘플만 교환. silver 훈련 암에서 선택적 채택.
- **지름길 학습(Shortcut Learning)** (Geirhos et al. 2020 `[요검증]`) — 모델이 진짜 인과 대신 상관 지름길(레이어 이름)을 탄다. **P2의 mask-ablation 암은 정확히 이 진단 도구**이며, 다이제스트의 H3(이름-prior 탑승 여부)를 직접 겨냥.

### 1.4 왜 Δ-ablation이 "본선"인가 — 계측 철학

P2가 기대는 것은 SOTA 추격이 아니라 **ablation(제거 실험)** 의 계보다. 프로그램은 이미 **셔플 대조군(shuffle control) AUC 0.375**(다이제스트)로 "무작위화하면 신호가 사라지는가"를 재는 규율을 갖고 있다. P2는 이 규율의 확장이다: 무작위화가 아니라 **문맥 채널을 통제된 방식으로 제거**했을 때의 붕괴량을 잰다. 이것이 "GNN이 좋더라"(FM4·검증 전 전향)를 피하고 "문맥이 없으면 얼마가 무너지는가"라는 반증 가능한 질문으로 바꾸는 장치다.

**가설 표기(패킷 문맥에서 추론 — H1/H2/H3의 원 정의는 패킷에 완전 기재되지 않음, `[추론]`):**
- **H1 [추론]**: 순수 결정론 기하(문맥 없음)만으로 벽 판별이 충분하다 (P1이 검증).
- **H2**: 관계적 문맥 피처가 벽 판별을 유의미하게 향상시킨다 (P2의 Δ-게이트가 생사 판정).
- **H3**: 탐지 가능한 "벽" 신호의 일부는 기하가 아니라 레이어명/관례가 나른다 (mask-ablation이 방향 증거 제공).

## 2. 알고리즘 정확 스펙

### 2.1 그래프 구성 G = (V, E)

**노드 V** — 도면 내 CAD 엔티티. 폴리라인은 세그먼트 단위로 분해(각 세그먼트 = 1 노드). 이는 CubiCasa SEG-IR의 진리 단위("Wall 클래스 요소의 **모서리**", 다이제스트)와 정합한다.

**노드 피처 x_v ∈ ℝ^d** (GBDT 6피처를 상위집합으로 승계):
```
x_v = [ onehot(entity_type)          # LINE / ARC / LWPOLYLINE-seg / SPLINE-근사 / ...
        log(length + eps),           # GBDT의 log길이
        sin(2θ), cos(2θ),            # 무방향 각도(주기 π) — GBDT와 동일
        thickness_evidence,          # 지역 평행-파트너 탐색의 두께 채널 스칼라
        parallel_evidence,           # 평행 채널 스칼라
        junction_evidence,           # 정션 채널 스칼라
        bbox_wh_norm(2),             # (선택, 좌표-누수 위험 — ablation 대상)
        e_layer ∈ ℝ^k ]              # ★레이어명 학습 임베딩 — MASK-ABLATION 축★
```
`e_layer`는 학습되는 임베딩(embedding table). **mask 조건 = e_layer를 0벡터로 고정**. 이 한 축의 on/off가 H3와 silver-누수 경보를 동시에 켠다.

**엣지 E와 엣지 피처 x_uv** — "wall_pairs 후보 그래프의 학습판"(패킷 mechanism):
```
평행 엣지 : |Δθ| ≤ 2°  AND  gap ∈ [50mm, 400mm]      # 탐지기 v1 허용치 승계
정션 엣지 : 끝점 거리 ≤ 6mm (snap)                    # 탐지기 v1 snap
오버랩 엣지: 공선 overlap ≥ 0.5                        # 탐지기 v1 overlap
x_uv = [ Δθ, gap, overlap_ratio, onehot(edge_type∈{parallel,junction,overlap}) ]
```
**차수 상한(degree cap)**: 후보 폭발(R12·quadratic)을 막기 위해 노드당 엣지를 gap 오름차순 top-K(예 K=8)로 절단하고, 잘린 엣지 수를 로그로 남긴다(무증거 truncation 금지 — 헌법 FM9). 좌표 단위 미상 도면(CubiCasa px)에서는 gap/두께 대역을 도면별 축척으로 정규화하거나, 축척 미상이면 **상대 대역**(치수-정박, 반대의견 #5 계열)으로 전환하는 스위치를 둔다.

### 2.2 GNN 순전파 (GraphSAGE 변형, L∈{2,3} 층)

```
h_v^0 = MLP_in(x_v)
for l = 1..L:
    m_v^l   = AGG_{u∈N(v)} ( W_edge^l · [ h_u^{l-1} ; x_uv ] )     # AGG ∈ {mean, max, attention}
    h_v^l   = σ( W_self^l · h_v^{l-1}  +  m_v^l )                  # σ = ReLU, + LayerNorm, + dropout
p_v = sigmoid( w^T h_v^L )                                         # P(wall-member(v))
```
- **관계형 변형(R-GCN 스타일)**: W_edge^l 을 edge_type별로 분리(W_par, W_junc, W_over) → 동종/이종 혼합 대응(§1.2). 기본 on/off를 하이퍼파라미터로.
- **GAT 암**: AGG를 attention으로: α_uv = softmax_u( LeakyReLU( a^T [ W h_v ; W h_u ; x_uv ] ) ).
- 층 수 L을 2~3으로 얕게 두는 이유: 벽 판별의 문맥은 **국소적**(수 홉 내 이웃)이라는 사전판단 + over-smoothing 회피. 깊이가 도움 되는지 자체를 셀에서 검증.

### 2.3 손실·클래스 불균형

CubiCasa 벽 선분율 **~11.8%**(다이제스트) → 강한 불균형.
```
가중 BCE : L = -Σ_v [ w+ · y_v log p_v  +  w- · (1-y_v) log(1-p_v) ],   w+ ≈ (1-π)/π,  π≈0.118
또는 focal loss (γ=2)      # hard example 집중
silver 암 : nnPU 또는 대칭 CE  # 노이즈-양성 대응 (§1.3)
```
**보고 지표는 PR-AUC 우선**(불균형에서 ROC-AUC는 낙관 편향). F1은 val에서 봉인한 임계 τ로 계산.

### 2.4 하이퍼파라미터 공간 (val에서만 탐색, test 단발)

| 노브 | 후보 |
|---|---|
| hidden dim | {64, 128, 256} |
| 층 수 L | {2, 3} |
| aggregator | {mean, max, attention(GAT)} |
| edge-type 관계형 | {off, on(R-GCN)} |
| dropout | {0, 0.2, 0.5} |
| lr / weight_decay | {1e-3, 3e-4} / {0, 1e-4} |
| 이웃 샘플 수(홉별) | [25, 10] |
| 손실 | {가중BCE, focal, (silver)nnPU} |
| **e_layer dim** | **{0(mask), 8, 16}** ← H3 축 |

**출력·추론**: 도면별 전체 그래프 inductive 순전파. τ는 val에서 봉인 후 test·실도면에 고정 적용.

## 3. 벽 과업 적응 설계 — 세 하네스축 접속

### 3.1 CubiCasa SEG-IR (벡터축) — P2의 본진 훈련·평가장

여기가 그래프가 **네이티브로 존재**하는 유일한 축이다(선분+관계). 데이터: train 4,200(선분 3.86M) / val 400(35.4만) / test 400(37.5만), 벽선분율 ~11.8%, 변환 실패 0, 레이어 중립(라벨 누출 0)(다이제스트).

**GBDT가 이미 여기서 0.517을 세웠다. P2가 그 위에 무엇을 더 가져오는가 — 기계론적 가설:**

GBDT는 **P 0.860 / R 0.370**. 정밀하지만 **벽의 63%를 놓친다**. 놓치는 벽은 십중팔구 *지역 증거가 약한* 벽이다: 평행 파트너가 가려졌거나(occlusion), 개구부(도어·윈도) 근처에서 끊긴 벽, 끝단 벽. **그래프 문맥 — "네 공선/정션 이웃이 전부 벽이고, 방 경계가 너를 관통해 이어진다" — 은 바로 이 놓친 벽의 재현율(recall)을 복구하는 신호다.** 따라서:

> **P2의 이득은 정밀도가 아니라 재현율 측에서 나올 것으로 예측한다.** ΔF1 ≥ +0.1이 관측된다면 그 대부분은 R 0.370을 끌어올린 결과여야 한다. 만약 상승이 정밀도에서만 나오면 그것은 문맥이 아니라 정규화 효과를 의심해야 한다(진단 신호).

이 예측은 반증 가능하다 — 셀에서 ΔR과 ΔP를 분해 보고하여, "문맥 = 재현율 복구"라는 인과 서사를 검증하거나 기각한다.

**"문맥 없는 최강 기하 P1"의 조작적 정의 문제 (중요한 정련):**
패킷은 P1을 "문맥 없는 최강 기하"라 부르지만, GBDT의 6피처는 이미 `parallel`·`junction`(쌍(pairwise)-유도 피처)을 포함한다 — 즉 GBDT조차 완전한 "문맥 없음"이 아니다. 그래서 Δ의 분모를 명확히 못 박는다:
- **엄밀 P1 = 노드-내재 피처만**(길이·각도·타입, 쌍 피처 제외)로 학습한 분류기 → "진짜 문맥 없음".
- **실용 P1 = GBDT 6피처(수작업 쌍 피처 포함) = 0.517** → "수작업 1차 문맥까지 포함".
- **P2(GNN) = 학습된 다중홉 문맥**.

따라서 Δ를 **두 층으로 분해**한다: Δ₁ = (실용 P1 − 엄밀 P1) = 수작업 쌍-피처의 기여; Δ₂ = (GNN − 실용 P1) = **학습된 다중홉 문맥의 순수 기여**. **P2의 프리레그 게이트(ΔF1≥+0.1)는 Δ₂에 건다** — 이것이 GNN이 정당화되는 유일한 지점이기 때문이다. 이 분해가 없으면 "문맥 기여"가 수작업 쌍 피처와 다중홉 학습에 혼입되어 판별이 흐려진다.

### 3.2 FloorPlanCAD (래스터축) — P2에서는 **약한 진리원, 정직하게 강등**

패킷 mechanism은 truth 3원을 synthetic / **FPC(transfer)** / silver로 둔다. 그러나 다이제스트가 명시하듯 **FloorPlanCAD 자산은 래스터 5,308장 + 벽 bbox/segmask이고 "벡터 SVG 없음"**. 벡터 GNN은 프리미티브 그래프를 요구하는데 FPC에는 프리미티브가 없다. FPC를 P2의 그래프 훈련원으로 쓰려면 래스터를 벡터화(선추출)해야 하고, 이는 그 자체로 오류를 주입해 "깨끗한 진리" 전제를 무너뜨린다.

> **정직한 입장(pushback)**: FPC는 P2(벡터 GNN)의 3원 삼각측량에서 **약한 다리다**. P2의 실질 3원은 **synthetic / CubiCasa(사람 라벨) / E1.5 silver** 로 재정의하는 것이 옳다. FPC는 벡터-GNN의 진리원이 아니라 **CL-G(래스터/VLM 트랙)의 교차대조 자산**으로 재배치한다. 이 재정의를 P2 계약에 명시한다(패킷 자기서술의 수정 지점).

### 3.3 1.dwg 실도면축 (384 def, silver) — 진짜 타깃 도메인

최종 타깃. 도메인 갭이 크다: 좌표 단위(실도면 mm vs CubiCasa px), 엔티티 혼합(실도면은 SPLINE 3,973/ARC 2,198/HATCH 264 혼재 vs CubiCasa/synthetic는 LINE/폴리라인 중심)(다이제스트). **이 갭 때문에 inductive GraphSAGE가 필수**(transductive는 전이 불가).

- **전이 성적의 현주소**: 순수 기하 탐지기 zero-shot이 CubiCasa val에서 **F1 0.2358**(다이제스트) — 약함. P2의 주장: CubiCasa 사람 라벨로 supervised 학습한 GNN이 (a) CubiCasa val에서 GBDT 0.517을 문맥으로 넘고, (b) 그것이 실도면 silver로도 전이되는가를 잰다.
- **silver 축과 P2의 위험한 교차점**: 다이제스트 B5 — 탐지기↔silver Pearson **0.2911**, name-blind 팔과 **완전 동일(full-vs-nb 1.0)** = 현 탐지기는 레이어명 신호가 0. 그런데 **P2의 GNN은 e_layer(레이어 임베딩)를 피처로 넣는다.** 즉 P2는 탐지기가 안 타던 레이어-이름 지름길을 탈 수 있다. silver 자체가 레이어명과 상관돼 있다면(E1 판정자가 이름을 탔다면), GNN이 silver 동의도를 **기하 이해 없이** 부풀릴 수 있다. → **mask-ablation 암이 이 누수의 탐지기**다. 마스크 시 silver 동의도가 붕괴하면 H3 증거인 동시에 silver-누수 경보(다이제스트 leakage 방지 조항 그대로).

**요약 — P2가 0.236(전이)과 0.517(GBDT)을 알고도 가져오는 것 3가지:**
1. **재현율 복구 가설의 검증**: GBDT가 놓친 63% 벽을 문맥이 얼마나 되찾는가(반증 가능).
2. **Δ₂ 계량**: 학습된 다중홉 문맥의 순수 기여치 — H2의 생사 숫자.
3. **H3 방향 증거**: mask-ablation으로 "신호가 기하인가 이름인가"를 실도면 silver 축에서 폭로. **음성 결과(Δ₂<+0.1)조차 값지다** — "현 계측기에서 문맥은 한계 기여"라는, 그래프/DL 사다리 전체를 강등하는 판결이기 때문.

## 4. 데이터·컴퓨트 요구

### 4.1 로컬 실행 계획 (RTX 5070 Ti 16GB / RAM 64GB — 이것으로 충분)

- **피처·그래프 빌드**: CPU + 64GB RAM. train 3.86M 선분이지만 **그래프는 도면 단위**(train 4,200장 → 도면당 평균 ~920선분). 엣지 구성은 도면별 top-K 이웃 → 차수 상한으로 quadratic 회피. 1회 precompute 후 캐시(parquet/npz).
- **훈련**: 소형 GNN(hidden ≤256, 2~3층)은 이웃 샘플링으로 16GB에 **여유롭게** 적재. 최대 도면정의 **412,775 선분**(다이제스트, 연산 병목 실증)만 예외 — 이 괴물 그래프는 Cluster-GCN/GraphSAINT식 서브그래프 파티셔닝 또는 도면-내 타일링으로 분할 처리. VRAM은 이 규모에서 제약이 아님.
- **시간(대략, 자원 계획용 — 측정 아님)**: 피처 빌드 수 시간; 4,200장 1 epoch 수 분; 전체 스윕 1~2일 로컬.

### 4.2 DGX 계획 (분리 — 임계경로 아님)

DGX Spark(Ornith-35B)는 현재 **unreachable(승인은 됨)**(다이제스트/부트). **P2는 DGX에 의존하지 않는다.** DGX 야간 슬롯은 오직 **145장 modelspace 전체 그래프로 확장**할 때만(vLLM Ornith 서빙과 시간분할, 체크포인트 재개 전제). 이 확장은 선택적 후속이며, val/test 판별의 임계경로 밖. DGX가 계속 불통이어도 P2의 본체(CubiCasa 벡터축 Δ-실험)는 100% 로컬에서 완주 가능.

### 4.3 데이터 가용성 점검

- CubiCasa SEG-IR: 이미 변환 완료(실패 0). 재사용.
- silver: E1.5 판정자 5기 → 실은 **~2어휘 가문**(fable+sol vs opus+sonnet+grok, 다이제스트) → "5독립" 아님. silver 훈련 암은 **상위 판정자 만장일치 def만** 사용(패킷 expected failure modes). 게이트 미충족 시 silver 암 비활성.
- synthetic: **아직 존재하지 않음**(PR-1/T2 — 생성기 부재). P2 synthetic 암은 CL-C 완성에 의존(§7).

### Cell 0 — IR 인접성 완전성 감사 (하드 선결)
- **가설**: Graph IR이 진짜 벽-인접성을 충분히 담아, 메시지패싱이 의미를 갖는다.
- **지표**: gt 벽-벽 인접쌍 중 그래프 엣지로 존재하는 비율; 차수 분포; L홉 내 벽-이웃 도달률; 연결성분 통계.
- **제안 합격선**: gt 벽-벽 인접의 ≥0.90이 2홉 내 도달(봉인 대상).
- **킬 조건**: 인접 재현 < 0.70 → 그래프가 과소연결 → GNN 무의미 → **훈련 착수 전 STOP**(IR 수정은 CL-B 소관으로 반송).
- **예산**: 로컬 CPU, <1일. **시드**: 결정론(무시드).
- **정당화**: R23(인접성 완전성 미증명, disputed)의 직접 응답. 이 감사 없이 훈련하면 GNN의 실패가 "문맥 무익"인지 "그래프 결손"인지 구분 불가.

### Cell 1 — Occam 사다리 하단 + P1 앵커 고정
- **가설(H1 경계)**: 선형 불충분, GBDT가 강한 지역 기저(실용 P1).
- **지표**: val F1 / PR-AUC. 앵커: logistic **0.053**, GBDT **0.517**(다이제스트 — **재현이지 재측정 아님**).
- **제안 합격선**: GBDT를 0.517 근방으로 재현 → Δ의 분모 잠금. 엄밀 P1(노드-내재만)도 함께 측정해 Δ₁/Δ₂ 분해 준비(§3.1).
- **킬 조건**: GBDT 재현 실패 → 피처 파이프라인 결함 → STOP.
- **예산**: <1일(대부분 존재). **시드**: GBDT 5시드로 기저 분산 기록.

### Cell 2 — 비학습 이웃-집계 GBDT (가난뱅이 메시지패싱)
- **가설**: 1홉 이웃 피처의 mean/max 집계를 GBDT에 먹이면, GNN 없이도 "문맥"의 대부분을 얻는다.
- **지표**: ΔF1 vs Cell 1(실용 P1).
- **제안 합격선/판정**: 이 단독으로 ΔF1 ≥ +0.1 → 문맥은 돕지만 **GNN 불요**(Occam → GNN은 "1홉 집계 대비 무-한계가치"로 강등).
- **킬 조건(GNN 측)**: Cell 2 ≈ Cell 3(GNN)이면 GNN은 1홉 집계 이상을 못 준다.
- **예산**: 1일. **시드**: 5시드.
- **정당화**: 패킷 cheapest-probe 조항(Occam 사다리를 비용순 강제)의 이행. **P2가 스스로를 죽일 수 있는 가장 값싼 칼.**

### Cell 3 — GNN 본 Δ-실험 (CubiCasa val) ★본체★
- **가설 H2**: 학습된 다중홉 관계 문맥이 지역/1홉 피처를 넘어 벽 판별을 향상.
- **지표**: **Δ₂ = F1(GNN) − F1(실용 P1)**, 그리고 ΔPR-AUC, **ΔR / ΔP 분해**(재현율 복구 서사 검증, §3.1). 동일 def-스플릿 held-out.
- **제안 합격선(프리레그)**: **Δ₂F1 ≥ +0.1 → H2 지지 · [0,+0.1) → H2 demote(문맥 한계 기여) · <0 → H2 kill(현 계측기 하)**.
- **킬 조건**: Δ₂ < 0 (문맥이 P1보다 해로움).
- **예산**: 1~2일, RTX 5070 Ti. **시드**: **≥5시드, mean±sd 보고. Δ₂가 시드 노이즈(sd)를 초과해야 유효**(seed-confound T15 직접 응답 — Δ가 시드 산포에 묻히면 지지 주장 무효).

### Cell 4 — mask-ablation 암 (H3 + silver-누수 경보)
- **가설 H3**: 레이어/이름 피처가 (지름길일 수 있는) 신호를 나른다.
- **지표**: ΔF1(full) − ΔF1(masked, e_layer dim=0); **실도면 silver 축에서 full vs masked의 silver-동의도**.
- **판정**: 마스크 시 큰 하락 → 이름 지름길(silver-누수 경보); 작은 하락 → 기하 주도. **마스크 GNN이 GBDT(0.517) 이하로 붕괴하고 그 상승이 silver 축에서만이었다면 → Cell 3의 H2 지지 RETRACT**(문맥 이득이 실은 이름-누수였음).
- **킬 조건**: 진단 셀(단독 킬 없음)이나 위 retract 트리거 보유.
- **예산**: +0.5일(e_layer 0으로 재실행). **시드**: Cell 3과 동일 시드 세트.
- **정합**: 다이제스트 B5(탐지기 name-blind 완전 동일)의 **학습판 대응물**. 탐지기는 이름신호 0이었으나 GNN은 이름을 탈 수 있다 — 이 셀이 감시.

### Cell 5 — 교차도메인 전이 CubiCasa → 1.dwg silver (게이트)
- **게이트(정정 필요)**: 패킷 P2는 "silver 암 활성 = E1.5 **B1 ≥ 0.70**"이라 인용하나, 반대의견 원장 #1에 따르면 **prereg_e15.json 기준 calibration이 정확**: B1 = well-posed(문제 성립) 게이트, **B4 Pearson ≥ 0.70 = silver 자격**. → **본 셀은 정정된 게이트(B1 well-posed AND B4≥0.70)에서만 활성화**한다. P2의 B1-단독 인용은 오류로 인정·수정(§7 T10 부수).
- **가설**: CubiCasa-훈련 GNN이 실도면 silver로, 현 탐지기(zero-shot)보다 잘 전이.
- **지표**: 실도면 held-out def에서 silver 동의도(Pearson/F1), name-blind 팔 포함. 기저: 탐지기 Pearson **0.2911**.
- **제안 합격선**: 0.2911을 봉인된 마진 이상으로 상회.
- **킬 조건**: ≤ 0.2911(현 탐지기 대비 무이득) → 전이 암 사망.
- **예산**: 게이트 해제 후 +1~2일. **시드**: 다수. **def-단위 스플릿, silver-훈련 def ≠ 평가 def**(누수 방지).

### Cell 6 — synthetic-훈련 → FPC/실도면 (보조가정, H2 본체와 분리 기재)
- **가설(보조)**: synthetic 분포가 전이 가능한 GNN 훈련에 충분하다.
- **지표**: synthetic-훈련 GNN vs 결정론 베이스라인 (FPC는 벡터화 필요 → 오차 주입 명시, 또는 실도면 silver).
- **제안 합격선/킬**: 결정론 베이스라인 미달 → **"synthetic 분포 충분" 보조가정 KILL**(H2 본체 아님 — 별도 원장).
- **예산**: 1일(단, synthetic 생성기 CL-C 존재 후). **시드**: 다수.
- **정직한 사전예측**: B1 충실도 FAIL(KS **0.5792**, TV **0.265**)에 synthetic이 3종 엔티티뿐(다이제스트)임을 고려하면 **이 보조가정은 실패할 공산이 크다**. 그럼에도 실행하는 이유: 실패가 "합성 3원 다리는 벡터 GNN에 부적격"이라는 값진 판결이기 때문(진리원 삼각측량의 한 다리를 정직하게 끊는 증거).

### 최종 test 단발
- val 전 개발 종료 후, **선택된 단일 구성**을 CubiCasa test(400장, 무접촉) + (게이트 해제 시) 실도면 test def에 **1회** 적용. 셔플 대조군·0벽/전벽 sentinel·recall 최저선 탑재. 봉인 밴드. **재튜닝 금지**(단발 원칙).

## 8. 인접 제안과의 관계

### 8.1 병합 지점 (홈 클러스터 CL-F)

P2는 **CL-F(학습 사다리: 고전ML→PU→GNN) = platt P2 + calibration P2/P3 + doe P1**의 GNN 상단이다. 독립 구축이 아니라 **하나의 사다리 연구로 병합** 실행한다:
- **calibration P2/P3** — "P1 대비 lift" 밴드와 PU 변형을 공급. P2의 Δ-게이트와 동일 골격 → 밴드 정의를 공유(단일 프리레그).
- **doe P1(마스터 스크린)** — representation × truth 요인 스크린. **단, PARKED 상태**(family×self-training confound + seed-confound T15). P2 Cell 3의 시드 규율이 doe P1 재개의 선결 증거를 일부 공급.
- 실행 형태: `probe_ladder.py` 하나가 logistic(0.053)→GBDT(0.517)→이웃집계GBDT→GNN을 **한 표·한 스플릿·한 시드세트**로 겨룬다. 사다리 각 칸의 한계 이득이 곧 판별.

### 8.2 차별점

- **vs P1 / CL-B (결정론)**: P1은 P2의 **분모**다. P2는 P1을 대체하지 않고 P1 위의 문맥 기여를 잰다. P1이 없으면 P2는 해석 불가.
- **vs calibration(정확 게이트)**: silver 자격 게이트에서 P2의 원 인용(B1)이 틀렸고 calibration(B4)이 맞다(반대의견 #1) → **이 점은 calibration을 따른다**(R9: 더 검증된 쪽 채택, 평균 금지).
- **vs CL-E (진리원 교차요인)**: CL-E는 동일-def 3원 불일치 구조를 잰다. P2는 그 결과를 **소비**한다 — 어느 진리원을 신뢰할지의 가중을 CL-E 독립성 결과로 정한다.
- **vs CL-K (anti-silver 통제)**: CL-K의 "gate-only vs silver-distill 1-epoch 대조"를 P2 Cell 4/5의 **상설 동반 통제**로 편입(반대의견 #2의 실험적 보존).
- **vs feyerabend P1 / CL-J (room/face-first)**: **P2의 가장 깊은 라이벌**. CL-J는 "벽을 노드로 분류"하는 관측 언어 자체를 뒤집어 "방/면(face)이 먼저, 벽은 dual의 bridge"라 주장. 만약 CL-J의 face-first 프로브가 messy divergent-20에서 P2를 강하게 이기면, **P2의 노드-분류 프레이밍이 틀린 관측 언어**라는 뜻이 된다.

### 8.3 이 제안이 죽어야 하는 조건 (정직하게)

1. **Cell 0 실패**: IR 인접 재현 <0.70 → 메시지패싱 무의미 → IR 수리 전까지 P2 사망(문제는 CL-B로 반송).
2. **Cell 2 ≈ Cell 3**: 1홉 이웃-집계가 문맥의 전부 → GNN은 무-한계가치 → P2를 "GBDT 피처엔지니어링 각주"로 축소(Occam).
3. **Cell 3 Δ₂ < 0**: 문맥이 P1보다 해로움 → **H2 kill**, 그래프/DL 사다리 강등.
4. **Cell 4 retract 트리거**: 마스크 GNN이 GBDT 이하로 붕괴 + 상승이 silver 이름-축에서만 → P2의 이득은 누수였음 → 지지 회수.
5. **P1(CL-B) 미통과**: 해석 가능한 분모 부재 → P2 판별 불능 → PARK.
6. **PR-3 라이선스 미해결**: CubiCasa 본진 봉쇄. synthetic(Cell 6)마저 B1 FAIL로 실패하면(예측됨) P2는 훈련 기질을 잃어 **심각히 약화**.
7. **CL-J face-first 압승**: 관측 언어 오류 판정 → P2 대체(superseded).

**판별 기여 요약**: P2는 **H2 생사(Δ₂-게이트) + H3 방향 증거(mask 암) + 고전ML/DL 사다리 커버리지**를 동시에 제공한다. 이 셋 중 어느 것도 P2가 "이기는" 것을 전제하지 않는다 — 지든 이기든, 사다리의 한 칸을 결정적으로 판정하는 것이 기여다. **P2의 가장 값진 산출이 "문맥은 현 계측기에서 +0.1을 못 넘는다"라는 음성 판결일 수도 있다는 사실을, 이 도시에는 정직하게 받아들인다.**
