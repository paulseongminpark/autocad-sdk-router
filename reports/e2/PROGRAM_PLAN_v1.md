발주: Paul — "1~6 모두 한다. 플래닝 해봐."
전제: 사람 라벨 0 · 원본 CAD READ-ONLY(staging만) · 외부셋 연구용 사용 GO(Paul 07-17) ·

규율: 실험마다 사전등록(합격선을 실행 전에 봉인) · 증거 xlsx 의무 · disk-first RUN_STATE · no fake PASS.

## 0. 목표 한 줄

도면 블록에서 벽을 사람 라벨 없이 찾아내는 탐지기 가족을 만들고, 그 성능을
4종 정답 대용(합성정답 · 외부셋 라벨 · 불변성 검사 · 상위 LLM 합의 silver)으로 정직하게 계측한다.

## 1. 전체 그림 (의존 순서)

```
[S1 법의학 감사]──┐
[S2 합성 생성기]──┼─(G1: 계측 신뢰 확보)──[S4 탐지기 v1 보수]──┐
[S3 외부셋 조달]──┘                                            ├─(G2: 평가팩 동결)──[S6 학습 사다리]
                              [S5 불변성 배터리]───────────────┘
```

- S1·S2·S3은 서로 독립 → **병렬 발사** (Wave 0).
- S4는 S1의 판정(무엇이 착시였나)과 S2의 픽스처를 입력으로 받는다.
- S5는 S4의 탐지기가 있어야 의미 있게 돌지만, 배터리 코드 자체는 병렬 제작 가능.
- S6은 G2(평가팩 동결) 이후에만 — 비싼 실험을 흔들리는 자 위에 올리지 않는다.

## 2. 단계별 실험 카드

### S1 — E1 법의학 감사 (패널 CL-A) · 로컬 CPU, 소규모

- **목적**: 어제 E1의 "AI-탐지기 불일치 top-20"이 실체인지 계측 착시인지 판별.
  의심 3건: ① v0 탐지기가 LINE만 읽고 INSERT(블록 속 블록) 미전개 ② top-20 단일종은
  정렬키 설계의 산물 ③ ornith 벽핸들 17/20건=정확히 10개는 "최대 10개 나열" 지시의 아티팩트.
- **입력**: bench/e1_shards projection + raw 판정 100파일 + calibration_v0/v1.
- **절차**: 결정론 스크립트 1개 — 인용 핸들 실재성 검증, def별 엔티티 히스토그램,
  INSERT 깊이 계측, bbox 단위 감사, 정렬키 교란 재계산(3가지 대안 정렬로 top-20 재추출).
- **합격선(초안, 착수 시 봉인)**: 의심 3건 각각에 CONFIRMED_ARTIFACT / REAL / MIXED 판정 +
  "진짜 divergent def" 정제 목록 산출.
- **산출**: tools/e2_forensic_audit.py + reports/e2/s1_forensic_v1.{json,md,xlsx}.

### S2 — 벽 합성 생성기 (패널 PR-1→CL-C) · 로컬 CPU

- **목적**: 정답을 알고 시작하는 연습 도면. 이후 모든 학습·평가의 1차 채점기.
  (기존 synthetic_truth.py는 치수 전용 — 벽 코드 0줄, 신규 제작.)
- **절차**: ① 벽 문법 정의(평행 이중선·교차부·개구부·두께 분포는 실측 145장 통계에서 표집)
  ② DXF 생성기(ezdxf 경유 — 라우터 규율 준수) + per-handle 정답 원장(wall_member(h) ∈ {0,1})
  ③ **충실도 게이트**(red team T2): S1이 정제한 실 divergent def의 실현상(폴리라인 벽,
  블록 중첩, 비평행 조각)을 재현하는지 — 실도면 통계 vs 합성 통계 분포 대조.
- **합격선(초안)**: 충실도 게이트 통과 + 난이도 3계열(S 단순/F 실전형/M 변이) × 각 N장 팩.
- **산출**: tools/e2_wall_synth.py + assets 아님·staging에 팩 + reports/e2/s2_fidelity_v1.xlsx.

### S3 — 외부셋 조달·변환 (연구용 GO) · 로컬, 대역폭만

- **목적**: 남이 라벨해 둔 정답 도면을 학습·평가 재료로 편입.
- **대상**: FloorPlanCAD(벡터 CAD + 선 단위 클래스 라벨) 우선, CubiCasa5K(래스터+폴리곤) 차순.
- **절차**: ① 공식 배포처에서 다운로드(→ D:\mirror 존, 외부 SoT 사본 규약) ② 인벤토리
  (장수·라벨 분포·포맷 실측) ③ **변환기**: 그들 포맷 → 공통 중간표현
  JSONL {segments:[{pts, layer, label}]} → 우리 per-handle 평가·학습 피처로 잇는 어댑터
  ④ 라벨 어휘 매핑표(그들 'wall' 계열 ↔ 우리 벽 정의) 명문화.
- **합격선(초안)**: 파싱 성공률 ≥95% 실측 + 매핑표 왕복 검증(표본 수동 대조는 라벨링이 아니라 검산).
- **산출**: tools/e2_extset_ingest.py + reports/e2/s3_inventory_v1.{json,xlsx}.
- **잔존 제약 한 줄**: NC 라벨로 학습한 가중치는 제품 탑재 시점에만 재검토(연구는 무제약).

### S4 — 결정론 탐지기 v1 보수 (패널 CL-B) · 로컬 CPU

- **목적**: 직선(LINE)만 보던 v0의 눈을 도면 전체로 확장 — 커버리지-완전 결정론 탐지기.
- **절차**: ① LWPOLYLINE/MLINE/ARC 정규화 ② INSERT 월드좌표 전개(깊이 재귀)
  ③ 단위 정박(mm 절대대역 → 치수-정박 상대대역, 반대의견 #5는 A/B로 선판별)
  ④ 정션(교차부) 후필터 ⑤ 다중 증거 격자(평행쌍·두께·연결성·레이어 신호 분리 보고).
- **합격선(초안)**: S2 합성팩에서 per-handle precision/recall 밴드(봉인 시 수치 확정) +
  v0 대비 zero-detection def 비율 감소를 계량(현 zero_frac 0.682 기준선 — calibration_v0).
- **산출**: tools/wall_detect_v1.py + reports/e2/s4_detector_v1.xlsx.

### S5 — 불변성(metamorphic) 배터리 (패널 CL-D) · 로컬 CPU

- **목적**: 라벨 0으로 145장 전체에 적용 가능한 유일한 공용 심판 — "옳은 탐지기라면
  도면을 회전·이동·배율·단위변환·블록해체·레이어개명해도 같은 벽을 찾아야 한다."
- **절차**: 변환 6종 × 145장 staging 사본 생성 → 탐지기 실행 → 판정 불변율 계측.
  **의무 안전장치**(red team T7): 0벽/전벽 센티널 — "아무것도 안 찾는 탐지기"는
  불변율 100%로 통과해버리므로, 최소 재현율 하한을 함께 요구.
- **합격선(초안)**: 변환별 불변율 밴드 + 센티널 2종 통과.
- **산출**: tools/e2_metamorphic.py + reports/e2/s5_invariance_v1.xlsx.

### S6 — 학습 사다리 (패널 CL-F/H/G/K) · 로컬 GPU → DGX

착수 조건 = **G2**: S2 합성팩 + S3 외부셋 + E1.5 silver(병합-어휘 기준) + S5 배터리로
평가팩 **WSD-EVAL-v1을 동결**(이후 실험은 전부 이 팩으로 채점 — 팩을 흔들면 비교가 죽는다).
동결 직전에 **대리 독립성 감사**(패널 PR-2/CL-E, red team 최고 severity T1): 같은 def에서
4종 정답 대용의 불일치 구조를 계측 — 4개가 한통속이면 "여러 개 통과"가 확증이 아니므로,
독립성 수치를 밴드 해석의 가중치로 명문화한다.

사다리 (아래가 싸고, 위는 아래를 이겨야 자격):

1. **고전 ML** (로컬 CPU/GPU): 기하 피처(평행쌍 거리·길이·정션 차수 등) → 로지스틱/GBDT.
   **이름-차단 쌍둥이 arm 의무**: 레이어·블록명 피처를 뺀 버전을 항상 병행 —
   "WALL이라는 글자를 읽은 것"과 "벽 기하를 본 것"을 분리 계량(동어반복 방지).
2. **PU 학습** (로컬): silver 양성 + 미라벨 나머지로 positive-unlabeled 학습.
3. **GNN** (로컬 RTX → 초과 시 DGX): 도면을 그래프(선분=노드, 인접=엣지)로. 착수 조건 =
   1단이 천장을 보였을 때만(Occam 사다리 — 로지스틱이 이기면 GNN 불요).
4. **RL 자리 판별** (로컬, 학습 0): 벽 "집합 조립" 과업에서 greedy vs beam 탐색의 보상지형
   프로브 — greedy≈상한이면 **RL 훈련 없이 kill**(증거로 종결, 교리 아님). beam이 크게 이기면
   RLVR 파일럿(검증기 오수락률 ≤0.01 선계측이 사활 게이트).
5. **VLM 이중 트랙** (소액 API + DGX): ① 프런티어 최고 지능 VLM을 배심원으로 — divergent def
   렌더 1패스 교차대조(이름-맹 판정) ② 로컬 오픈 VLM DGX 파인튜닝은 ①이 신호를 보일 때만.
6. **anti-silver 통제 arm** (패널 CL-K, 상설): silver로 학습한 모델 vs 게이트만으로 학습한 모델
   대조를 사다리 전 단계에 병행 — "LLM 편향을 학습해버렸는가"의 실험적 보존.

- **산출**: 단별 prereg + xlsx + reports/e2/s6_ladder_*.md. DGX 배치는 RAM/VRAM 실측 초과 시.

## 3. 관문 (Gate)

- **G0** (Wave 0 착수): 이 플랜 승인.
- **G1** (S4 착수): S1 판정 완료 + S2 충실도 게이트 통과 + S3 인벤토리 완료.
- **G2** (S6 착수): WSD-EVAL-v1 동결 + 대리 독립성 감사 수치 확보.
- **G3** (사다리 승급): 각 단은 아래 단을 평가팩에서 이겨야(사전 봉인 밴드) 위로.

## 4. 웨이브 편성

- **Wave 0 (즉시, 전부 로컬·병렬)**: S1 + S2 + S3. S5 배터리 코드 골격도 병행 제작.
- **Wave 1**: S4 + S2 팩 확장 + S5 실행(145장).
- **Wave 2**: 독립성 감사 → WSD-EVAL-v1 동결 → S6-1/2 (고전 ML·PU).
- **Wave 3**: S6-3/4/5 (GNN·RL 판별·VLM) — DGX·소액 API는 이 웨이브에서만.

## 5. 리스크 → 대응 (red team 티켓 승계)

- T1 대리 한통속(sev 0.75) → G2의 독립성 감사 의무화.
- T2 합성 충실도 → S2 게이트를 실도면 통계 대조로.
- T7 0벽 탐지기 통과 → S5 센티널+재현율 하한.
- 정렬키 착시(T3) → S1에서 대안 정렬 3종 재계산.
- 동어반복(이름 학습) → S6-1 이름-차단 쌍둥이 arm 상설.
- 컴퓨트 폭주 → 후보 폭발 상한·타임아웃을 각 카드 prereg에 명기(로컬 LLM 폭주 4중 방어 판례 준용).
