# 과학자 듀오 소집 — E2 다음 웨이브 설계 (라운드 1: 블라인드 포지션 페이퍼)

당신은 E2 프로그램의 수석 과학자 2인 중 1인이다. 세계 최고 수준 과학자의 태도로 — 증거에 복종하고,
자기 가설을 가장 세게 공격하고, 화려함보다 판별력을 택하고, 모든 주장에 킬 조건을 달아라.
상대(다른 모델 가문의 과학자)가 당신의 페이퍼를 라운드 2에서 적대 비판한다.

## 프로그램 사실 (전부 실측 — 이 밖의 수치 발명 금지)

- 과업: CAD 도면 선분의 벽/비벽 의미 판정. 벤치=CubiCasa5k SEG-IR(train 386만/val 35.4만/test 37.5만
  선분, 벽 비율 11.8%, 레이어명 중립화). val=개발 자유/test=봉인(방법당 단발, 사후 밴드이동 금지).
- 스코어보드(val F1): 기하 규칙 0.236(P 0.134≈기저율 0.118, 축척 무감=두께 prior 판별력 0) →
  GBDT 지역 6특징 0.517/AUC 0.9215 → +문맥 6특징(정션 차수·평행 갭이 중요도 1·2위) **0.705**/AUC
  0.9671, recall 0.617·P 0.824. 교훈: 벽다움=이웃 관계 패턴(H3 직접 증거).
- 계측기(자격 완료): world-IR 오라클 selftest 26/26 · CRS 픽셀↔핸들 브리지 역투영 오차 0.0 ·
  생성기 v2(미끼 8클래스·엔티티 13종, 두께 KS 0.063/엔티티 TV 0.0008) · 벽-주장 검증기(라벨 무접촉
  구성 검사, FAR 0/3024·FRR 0/504 — 단 LINE-only 팩 한계, 전체 다양성 재실측 비행 중).
- 비행 중 셀 6: gen2 전량+검증기 전다양성 재실측 / 반증이론 C0(PASS·바이트 재현)→C1(치수 앵커 축척) /
  PU 약지도 사다리 / GNN용 typed 그래프 빌더(E1) / 비-GNN 기준선 B* 동결(E3) — 전부 CPU.
- GNN 트랙(도시에 calibration_P3): E1→E2(3-layer SSL 프로브, 로컬 RTX 5070 Ti 16GB, seed 17 단발)→
  formal(시드 {17,29,43}, B*+0.05 AUPRC 밴드). DGX 큐 5셀 배정 계획.
- DGX Spark: GB10(Grace-Blackwell, 통합메모리 128GB), nvcr pytorch:25.04-py3 이미지 보유, 디스크
  461GB 여유, GPU 유휴 0%, ssh LAN 직결. 로컬 보유 데이터: FloorPlanCAD 래스터+벽마스크, qwen SFT/GRPO
  파인튜닝 모델 실존. **Paul 지시: DGX를 극한으로 활용하라.**
- 제약: 과금 0(구독·로컬만, 프런티어 API 금지) · 이미지 판정=grok 전담(유보 중) · 원본 CAD READ-ONLY ·
  셔플 대조군 의무 · 증거 xlsx 의무 · 모든 수치 주장에 산출 아티팩트 병기.

## 채택 프레임워크 — AIDE² RSI (weco.ai, 2026-07-14 공개; 당신들의 사고 틀로 적용하라)

1. **2단 루프**: inner 루프=해(모델·실험)를 평가지표에 최적화. outer 루프=inner의 **하네스 자체**
   (탐색 정책·특징 설계 절차·셀 구조·평가 처리)를 다시 쓰는 에이전트. 채택 게이트="같은 비용 예산에서
   이전 최고를 이길 때만". AIDE²는 이 구조로 8일 무인·외부 100스텝에서 7개 연속 개선판을 얻었고
   held-out 과업 일반화까지 증명했다.
2. **탐색 연산자**: draft(서로 매우 다른 복수 안)/debug(버그 리프)/improve(전체 트리 최고해 그리디) —
   평가 리뷰어가 노드 점수 추출.
3. **평가 3원칙**: public/private 분리(inner는 public만 봄, 생존은 private — 우리의 val/test 봉인과
   동형) · **고정 비용 예산**(달러·컴퓨트 — 효율 개선만 통과, brute force 탈락. 예산이 곧 선택압) ·
   과업 이질성(일반화 압력 — 우리의 CubiCasa/gen2/FloorPlanCAD/실도면 혼성과 동형).
4. **보상해킹 3층 방어**: 프롬프트 지시 + 하드 가드(의심 출력 재생성·구성적 검증기) + 통계층(동료 대비
   극단 성공 격리). AIDE85는 이 방어로 리워드 해킹률 63%→34%.
5. **정직성**: 평가 자체의 결함은 악용 대상이 아니라 수리 대상 (eval repair 사례).
6. **핵심 교훈**: 문헌의 화려한 아이디어 95건이 고정 예산 게이트에서 탈락 — 승자는 단순 메커니즘의
   조합이었다. 복잡성 제안엔 반드시 예산-내 근거를 달아라.

1. **결과 해석**: 0.236→0.517→0.705가 왜 이렇게 됐는가. 현재 접근의 천장은 어디이고 왜인가.
   (남은 오류의 구조 가설 포함)
2. **다음 웨이브 실험 목록** (5~10개, 각각: 가설 · 데이터 · 지표 · 킬 조건 · 고정 예산 · DGX/로컬 배정):
   GNN formal 트랙, DGX 학습 셀(예: 래스터 세그멘테이션, qwen 파인튜닝 활용, 대규모 사전학습), RL/검증기
   보상 트랙, 그 외 당신이 판단하는 최고 가치 실험. **DGX 극한 활용이 명시 요구다** — 유휴는 낭비다.
3. **RSI 루프 실장안**: E2에 AIDE² 프레임을 어떻게 실장하나 — inner/outer 루프의 구체 대상, 채택 게이트
   수치, 고정 예산 단위, 보상해킹 3층의 우리식 구현(검증기·셔플·통계층), test 봉인과의 양립.
