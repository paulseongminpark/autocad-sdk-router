## 1. 결과 해석 — 0.236 → 0.517 → 0.705 (승계 + 2 수정)

### 1.1 세 숫자의 인과 해부 (승계, 어휘 수정)

라운드 1의 해부를 유지한다: **정보 추가의 역사가 아니라 "조합 학습(H2) → 관계 표현(H3)"으로 병목이 이동한 역사**다. 0.236은 판별력 0 술어의 시대(P 0.134 ≈ 기저율 0.118, 축척 스윕 불변 = 물리 두께 prior 사망; `R:PROGRAM_JOURNAL.md` 3장). 0.517은 조합의 시대(같은 특징으로 선형 F1 0.053 vs GBDT 0.517261 — 신호는 여백이 아니라 상호작용에 산다; `R:cells\cml_ctx\REPORT.md`, `R:PROGRAM_JOURNAL.md` 4-1장). 0.705는 관계의 시대(문맥 6특징이 F1 +0.188, 순열 중요도 1·2위가 정션 차수 0.0862·평행 갭 0.0741; `R:cells\cml_ctx\REPORT.md`). 누출 방어선(셔플 AUC 0.375/0.332, name-blind 쌍둥이 0.9315/0.9252, 좌표·handle·layer 미포함)도 승계.

**어휘 수정(자기 수정 ⑥):** 라운드 1에서 "회수율 0.370이 이 시대의 상처" 류의 운영점 의존 서사를 썼다. threshold 0.5의 FN/FP 카운트와 P/R은 운영점 선택의 그림자를 포함하므로, 이후 **표현-한계 논증에는 threshold-free 지표(AUPRC·PR 곡선)와 동정밀도 회수율만 쓴다**. 이 규율은 상대의 강주장 B(FN>FP → recall 병목)에도 동일 적용된다 — 방향은 동의하되 증거는 W2-01의 PR 분해로 교체돼야 한다.

### 1.2 남은 오류의 구조 가설 (승계 + 산출물 계약 추가)

E-H1(무짝·부분짝 FN) · E-H2(주석 밀집 FP) · E-H3(짧은 선분 FN) · E-H4(스타일 꼬리) · E-H5(라벨 바닥) 유지. FP 전수조사가 0.236 시대에만 존재하고 0.705 모델에 대해 부재하다는 사실도 유지 — sol의 H-R2 킬 조건(클래스 슬라이스)이 참조하는 측정 역시 존재하지 않으므로, **W2-01의 산출물 계약에 "0.236 시대 FP 클래스(화살표·경계폴리곤·치수·문·창) 기준의 0.705 오류 슬라이스 표"를 명시**해 쌍방 가설의 킬 조건이 소비 가능한 아티팩트가 되게 한다.

### 1.3 천장 — 사전 베팅으로 재표기 (자기 수정 ⑤)

**베팅: 고정 반경 스칼라 집계 + GBDT 가족의 val F1 천장은 0.75±0.03.** 근거 세 갈래는 라운드 1 그대로다(R-스윕 단조 감소 0.700393/0.689707/0.675017 · 중요도 꼬리 포화 <0.01 · 스칼라 집계의 구성 정보 원리적 손실; `R:cells\cml_ctx\REPORT.md`). **정직화:** 중심 0.75와 폭 ±0.03은 아티팩트에서 도출된 신뢰구간이 아니라 **설계된 베팅 폭**이다 — 상대의 "천장 숫자 무발명" 공격이 옳게 겨눈 지점이며, 표기를 바꾸되 베팅 자체는 유지한다. 킬 조건 달린 명시 베팅만이 "내가 틀렸다"를 판정 가능하게 하기 때문이다(주장 6).
**킬 조건(승계):** ① W2-02가 val-A F1 ≥0.78 → 기각(가족 정의 과소). ② W2-01에서 전 클래스 lift <1.5× ∧ 라벨 의심 >30% → 천장은 0.75보다 낮은 라벨 바닥에 있고 관계-모델 기대치 하향.

## 3. RSI 루프 실장안 v2 — AIDE² 프레임의 E2 이식 (합성판)

라운드 1의 골격(신설 3종: 청구 예산 장부·격리 val-B·통계층)을 유지하고, sol의 인터페이스·회계 장치를 접합한다.

### 3.1 inner 루프 — 해 최적화
- **대상(승계):** 1차 실장은 벡터 트랙 — 후보 = (특징 레시피, HGBDT 설정, 임계 정책). 고회전부는 한계비용 0인 CPU에 둔다(과금 0 프로그램의 AIDE² 변형). GPU/DGX 트랙은 소수 셀.
- **연산자(승계 + A9 흡수):** draft = 서로 다른 가설 계열의 복수안 — **같은 아키텍처의 미세 knob 다발은 draft가 아니다**. debug = 결함 수리 리프 — 점수 개선용 특징/HPO 혼입 금지, 평가기 결함은 outer로 승격. improve = 현 최고해에 **단일 메커니즘만 추가 + parent 대비 ablation 의무**.
- **INVALID 규칙(A9 흡수):** 예산 초과 또는 증거 xlsx/failure sheet 부재 시 점수 무관 INVALID.
- inner가 보는 것: val-A + gen2 개발 팩만. val-B·test 접촉 0.

### 3.2 outer 루프 — 하네스 재작성
- **대상(승계):** 특징 문법·탐색 정책 / 가드 집합 / 셀 구조·평가 드라이버 / 예산 배분 정책. `harness_vN` 버전·SHA 고정.
- **채택 게이트(승계 + 보강, 봉인 대상):** 동일 청구 예산의 replay에서 ① val-A AUPRC ≥ +0.005(하한) ② 전 가드 PASS ③ val-B 비열화 — **paired family-bootstrap CI 병기(sol 흡수)** ④ **더 엄격한 기존 게이트 우선(A7)**: GNN B*+0.05, M30 +0.10 등은 RSI 공통 게이트로 희석되지 않는다. 문턱 최종값은 **max(+0.005, 2×SE_boot(val-B))**로 봉인 — 상수가 아니라 W2-09가 실측하는 분산에서 절차적으로 유도한다(무발명 수치 규율의 실장).
- **주기(승계):** outer 채택 판정은 웨이브 경계 1회 — 하네스 재평가 질의가 val-B 질의 캡 안에 들어가게 하는 구조적 장치다.

### 3.3 고정 예산 — 통화의 벡터화 (A4 흡수)
청구 단위: **(CPU-h, RTX-h, GB10-h, peak RAM, disk I/O, val-B 질의 수, test 소비권=방법당 1, frontier 호출권=0)**. 축 간 환산 금지 — 남은 CPU로 GPU를 사지 않는다. challenger는 incumbent와 동일 상한 벡터. 모든 inner 후보는 실측 벽시계를 장부에 청구하고, 채택 비교는 동일 청구액에서만 성립 — brute force는 정의상 탈락. 웨이브 봉투는 §2의 표를 봉인.

### 3.4 보상해킹 3층 (승계 + 통계층 정의 보강)
| 층 | E2 실장 | 상태 |
|---|---|---|
| ① 지시 | 패킷 계약(CHANGE_ONLY·no-fake-PASS·상태 어휘 5종) | 가동 중 |
| ② 하드 가드 | 구성 검증기(name-blind 자격 포함 — 신규) · 센티널 팩 · 셔플 의무 · name-blind 쌍둥이 · fail-closed 오라클(26/26) · 증거 xlsx+SHA · 의심 출력 독립 재계산 | 가동 중, 검증기 재자격 비행 중 |
| ③ 통계층 | (a) 극단 성공 검역: 검역 통계는 **동세대+직전 세대 후보를 모집단으로, N≥8일 때만 z/P99 규칙 적용, N<8이면 전수 검역**(자기 수정 ④ — 소표본에서 MAD/백분위 퇴화). 해제 프로토콜 = 누출 스캔 + family-holdout 재검 + val-B 대조 + 독립 재계산(sol 흡수). (b) 절대 센티널: val-A AUC >0.995 또는 F1 >0.95 자동 검역(정직한 천장 ~0.71인 과업에서 준-오라클 성적은 누출 신호). (c) val-A/val-B 괴리 드리프트 모니터. | W2-09 신설 |

AIDE85의 63%→34%는 "3층을 깔아도 1/3이 남는다"로 읽는다(승계) — 최종 방어선은 층이 아니라 봉인된 test 단발 + frozen resolver다.

### 3.5 test 봉인과의 양립 (승계 + A8 흡수)
- public/private 동형: inner = val-A + gen2 개발 팩. private = test(F06 단발) + gen2 숨김 family + val-B(score-only·질의 장부·경계 집중). outer 채택 이력은 F06 진입 전 동결 — RSI가 test에 손대는 경로는 구조적으로 없다. test 접근은 별도 ACL·접근 카운터(항시 0 표시).
- 과업 이질성: inner 채점은 혼성 패널(val-A+gen2+가용 시 FPC 전이) — 단일 기질 승자는 "기질 특화"로 기록, 채택 안 함.
- eval repair(승계): 평가 결함은 수리 대상 — 영향 행 superseded 동결 후 수리 평가로 재실행, 알려진 결함 이용은 K07급. **(A8 흡수) test 이후 결함**: 밴드 이동·동일 method ID 재실행 금지, `INVALID_EVAL_DEFECT` 또는 원 판정 보존, 수리 평가기는 새 prereg·새 버전의 별도 거버넌스.
- 단순성 교훈(승계): W2-02(고전)와 융합(W2-04/08)이 W2-03(GNN)·A64(RL)보다 먼저다. 복잡성 제안은 동예산 고전 대조 의무(M30·B* 제도).

## 4. 강주장 — 갱신판 6개 (각각 킬 조건)

**주장 1 (유지).** 관계 문맥의 남은 이득 중 절반 이상은 GNN 없이 2-hop 고전 특징으로 회수된다 — GNN formal은 B*+0.05를 넘지 못할 확률이 더 높다. sol 강주장 A와 수렴하며, 판정 셀도 합병했다(W2-02). **킬:** W2-02 ΔF1 <+0.01 ∧ W2-03 SSL ablation family-holdout CI>0 → 주장 사망, GNN 주력 승격 동의.

**주장 2 (강화).** 현행 스코어보드는 val 적응 재사용 오염에 무방비이고, 0.705에는 상향 편향 가능성 주석이 붙어야 하며, **val-B 격리는 RSI 도입의 전제조건이다** — 이제 그 격리는 주기(경계 개봉)만이 아니라 **인터페이스(score-only)와 질의 장부(웨이브 캡)**를 갖춘 이중 방어다(sol과의 합성). 추가 논거: 질의 총량 회계 없는 score-only는 "천천히 public화되는 private"일 뿐이다. **킬:** W2-09 회고 팔의 A/B 델타가 CI 안 → "기왕 오염 무시 가능" 수용, 주석 요구 철회(전향 격리는 비용 ~0이므로 유지).

**주장 3 (유지).** DGX 극한 활용의 최고 가치는 큰 모델이 아니라 **전수(全數) 계산**이다 — 무샘플링 full-graph(412,775 seg), 전코퍼스 SSL, 시드 완주. **킬(강등 조항):** W2-00 처리율 <2× → "DGX = 메모리-바운드 잡 전용 큐"로 강등, throughput성 잡의 DGX 이관 철회.

**주장 4 (갱신).** RL 트랙은 **A63(무학습 진단) 또는 W2-07b(bandit 단)에서 죽는다**에 걸겠다 — 그리고 그 죽음은 프로그램의 성공 사례다. 라운드 1보다 죽는 지점이 하나 늘었다: sol의 사다리를 흡수해 "다단 구조가 있어도 bandit이 회수한다"가 2차 방어선이다. **킬:** A63 갭 ≥0.01 ∧ oracle 상한 갭 >0.01 ∧ A64 RLVR이 bandit 대비 utility ratio ≥1.05를 hidden semantic 비열등과 함께 재현 → 주장 철회, RLVR 지지(GRPO 선행 자산 비교 팔은 W2-05에 준비돼 있다).

**주장 5 (유지, 공동 주장 승격 제안).** 이번 웨이브 test 소비 0. sol도 동일 결론이므로 **웨이브 프리레그의 공동 조문**으로 승격을 제안한다. **킬:** 웨이브 종료 시 ≥2개 독립 방법 가족이 pre-test 게이트 전문을 통과하고 융합 설계가 동결되면, 차기 웨이브 선두 방법의 F06 개방에 동의.

**주장 6 (신설 — 방법론).** **킬 조건 달린 명시 베팅(폭 포함)이 천장 절제보다 낫다.** sol의 "숫자 무발명"은 틀리기를 회피하는 규율이고, 내 "0.75±0.03 + 킬 조건"은 틀림을 판정 가능하게 만드는 규율이다 — 반증가능성이 정보를 만든다. 단 폭은 도출된 신뢰구간이 아니라 설계된 베팅임을 병기해야 한다(자기 수정 ⑤). **킬:** 이번 웨이브와 차기 웨이브의 내 사전 베팅이 연속 2회 자기 밴드 밖으로 빗나가면, 이 주장을 철회하고 sol의 절제 규범을 채택한다.
