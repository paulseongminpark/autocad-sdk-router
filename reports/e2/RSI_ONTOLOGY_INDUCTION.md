# 온톨로지 귀납 + RSI 재귀 개선 — 설계·판단

작성 2026-07-20. 발주: Paul "온톨로지 실물을 더 낫게 하려면 GNN을 합성하면 되지 않나. 온톨로지
규약을 만드는 도구로서 GNN 같은 ML/DL/RL 모델을 사용하는 것. 그렇게 만든 n차 온톨로지 모델링을
또 한 번 RSI로 재귀 검증·improve." 상위: RSI_SYSTEM_DESIGN.md(@65880ec), M-14 결과(@7ccb3fc).

## 1. M-14가 왜 이 방향을 가리키는가 (근거)

M-14 실측: 손 규칙 16공리 단독 AUPRC 0.28, GNN 잔차 포획 0.58(K=8) — 그러나 **FP 포획 0.59 ≫
FN 포획 0.24**. 즉 손 규칙과 GNN이 **같은 기하 혼동을 공유**한다(둘 다 "벽처럼 생긴" 것에 속고,
둘 다 진짜 놓친 벽(FN)은 못 잡는다). 함의 두 가지:
- 손 규칙을 GNN에 더해도 상보성이 낮다(같은 곳에서 틀리므로).
- 손으로 쓴 공리는 **사람의 기하 직관 = GNN의 학습된 기하 직관**과 겹친다. 새 신호는 사람 직관
  밖에서 와야 하고, 그것을 찾는 도구가 바로 데이터에서 규칙을 **귀납**하는 모델이다.

## 2. 제안의 정식 이름과 방법론 가족

**온톨로지 귀납**: 모델을 분류기가 아니라 **규칙 발굴기**로 쓴다 — 데이터에서 해석 가능한
기호 규칙(온톨로지 공리)을 뽑아낸다. 후보 가족(전부 우리 42셀 사다리 안에 있던 계열의 재배치):

| 귀납 도구 | 산출 규칙 형태 | 성격 |
|---|---|---|
| GBDT(HGBDT) → 규칙 추출 | 결정 경로 = if-then 규칙 | 이미 0.75 in-sample 실측 자산 |
| GNN → 증류/모티프 추출 | 어텐션·메시지 패턴 → 부분그래프 모티프 | GNN 지식의 기호화 |
| ILP(귀납 논리 프로그래밍) | 1차 술어 규칙 | 고전 "온톨로지 귀납" 정통 |
| RL 규칙 탐색 | 규칙 제안→보상=holdout lift | 라벨-제로 제약과 궁합 |
| RuleFit/희소 규칙 앙상블 | 규칙+선형 가중 | 해석성+성능 절충 |

## 3. 목적함수 — 여기가 성패를 가른다 (내 핵심 판단)

**순진한 목적함수 = 함정.** "GNN을 도구로 규칙을 만들고 GNN 잔차로 채점"하면 **순환 보상 해킹**이
생긴다 — 귀납된 규칙이 그냥 GNN 출력을 암기해 버리면 잔차 포획이 올라가지만 새 신호는 0이다.
(weco reward-hacking 기본율 63%의 우리 판 — 반드시 방어.)

**올바른 목적함수 3항** (봉인 채점기, 비공개 truth 기준):
1. **단독 AUPRC** (비공개 분할의 **지상 truth** 대비 — GNN 일치가 아님). 규칙이 실제로 벽을
   맞히는가.
2. **상보성 = FN 가중 잔차 포획**. GNN이 **놓친 것**(FN, 현재 포획 0.24)을 얼마나 잡는가에
   가중. FP 재포획은 낮게 보상(이미 공유하는 혼동이므로 새 신호 아님).
3. **해석성 예산**: 규칙 수·복잡도 상한(온톨로지로 남으려면). 상한 넘으면 블랙박스 = 실격.

핵심 방어: **GNN은 "무엇을 우선할지"(잔차 표적)를 정의하는 데만 쓰고, 라벨로는 절대 쓰지
않는다.** 채점 truth는 항상 지상 truth. 이것이 순환을 끊는다.

## 4. n차 재귀 구조 (RSI 플라이휠의 온톨로지 판)

```
G0 = M-14 손 규칙 16공리 (봉인 SHA 0b51586b… — 기준선)
반복 k: 귀납기(§2 도구)가 Gk 규칙집 제안
      → 봉인 채점기가 §3 3항 목적함수로 평가 (공개/비공개 분할)
      → keep-if-better (비공개 지상 truth 기준) → 동결·SHA 봉인 = G(k+1) 기준선
      → 다음 반복은 G(k+1)에서 출발 (누적 개선)
종료: 목적함수 개선 정체(연속 무개선) 또는 예산 소진
```
- 이것은 M-13 RSI 하네스(C-3)의 **탐색 공간을 "온톨로지 규칙집"으로 특화한 인스턴스**다 —
  새 기계가 아니라 기존 하네스에 채점기 플러그(§3)와 후보 실행기(§2 귀납기)를 끼운 것.
- "재귀 검증" = 매 세대 규칙집이 봉인 채점을 다시 통과해야 하고(공개/비공개 분리로 과적합 차단),
  세대가 올라갈수록 규칙집이 지상 truth에 정직하게 수렴한다.

## 5. 3고리 모델과의 관계
- 이것은 **고리 1(탐지기 루프) 심화**다: M-13 봉인 탐색 공간의 "온톨로지 제약 규칙층"이
  손 작성(M-14) → **모델 귀납**으로 승격. M-13 봉인 목적(pooled AUPRC)은 불변이므로 이
  특화 목적함수(§3)는 **W4 신규 봉인 셀**(W4-onto-induct).
- 산출 규칙집은 해석 가능하므로 사람 리뷰(고리 3)와도 접속 — 귀납된 공리를 리뷰 좌석이 심사.

## 6. 위험·방어 요약 (pushback)
- **순환 보상 해킹**(§3): truth=지상, GNN=표적 정의 전용. 위반 시 런 무효.
- **해석성 붕괴**: 규칙 수 상한 없으면 온톨로지가 아니라 블랙박스 재발명. 예산 강제.
- **비공개 누수**: 귀납기가 비공개 분할·GNN의 비공개 예측을 보면 무효(C-3 권한 분리 승계).
- **세대 드리프트**: 각 세대 SHA 봉인·append-only 계보 — G0부터 추적 가능해야.

## 7. 편성 (W4 후보 — W3 로스터 불변)
- **W4-onto-induct**: 온톨로지 귀납 RSI 셀. 기준선 G0=0b51586b…, 귀납 도구 ≥2가족 비교,
  목적함수 §3 봉인, n차 재귀. C-3 하네스 재사용.
- 선행: C-3 하네스 착지(완료) + §3 목적함수(특히 FN-가중 상보성) 조작적 정의 선봉인 +
  귀납기 후보 코드 SHA 봉인.
- gen 방향(합성 데이터)과 무관 — 이건 실물 CubiCasa 위 귀납. M-11 청주 통과 시 둘째 프로젝트로
  일반화 검증 축 추가 가능.
