# M08H — native WRITE family: dimensions / annotations / hatch (staged-write)

Scope: turn `src/Ariadne.AcadNative/families/m08h_handlers.inc` from a stub into real
staged-write ObjectARX handlers. Single-file CHANGE_ONLY (`.inc` + this report + the unit
test). Seam (`m08hHasOp`/`m08hDispatch`) was pre-wired in `AriadneNativeJob.cpp`.

## Result summary
- Implemented: **12 / 15** ops (all feasible hostless).
- Deferred: **3 / 15** ops (style/content-bound; honestly left OUT of `m08hHasOp`).
- Native build: **exit 0**, `"status":"ok"` — all three modules link hostless.
- Source-level seam tests: **13 passed** (`tests/unit/test_m08h_handlers.py`).

### Hatch (1)
| op id | AcDb class | notes |
|---|---|---|
| `write.entity.hatch` | `AcDbHatch` | solid fill: `setAssociative(false)` + `setDatabaseDefaults(pDb)` + `setNormal(0,0,1)` + `setElevation(0)` + `setHatchStyle(kNormal)` + `setPattern(kPreDefined,"SOLID")` + `appendLoop(kExternal, vertices, zero-bulges)` + `evaluateHatch()`. Reads a `vertices` array of `>=3` `{x,y}` points (closed external loop). |

All handlers follow the canonical reference pattern (`appendLine` / `write.entity.line` in
`AriadneNativeJob.cpp`): `ensureLayer` (if a layer is named) -> open model space `kForWrite`
on the staged `ctx.pDb` -> `new AcDbXxx(...)` -> `setLayer` -> `appendAcDbEntity` ->
`close()`/`delete` correctly -> `countModelSpace` -> emit `"result":{...,"handle":...,
"modelspace_entities_after":N},"status":"ok"}`. Entity text routes through `utf8ToWide`
(lossless UTF-8 -> UTF-16; ASCII unchanged) so Korean/non-ASCII content survives; JSON
output routes through `njsonStr` (UTF-8). On any ObjectARX failure the handler emits a
structured `emitNativeError(...)` — never a fake `"ok"`.

## Deferred ops (3) — NOT in `m08hHasOp` (concrete blockers)

| op id | AcDb class | blocker (why not feasible hostless / honestly) |
|---|---|---|
| `write.entity.leader` | `AcDbLeader` | A valid leader is an *associated* annotation: it needs `appendVertex` of the leader line **plus** an annotation object (mtext/block/tolerance) bound via `setAnnotation`/`setDimensionStyle`. A bare leader with no annotation + no vertices is a degenerate entity, not a real "create a leader" op. Building a *meaningful* one requires an annotation-construction contract beyond a single point list — out of scope for this WRITE-primitive slice. Deferred rather than emit a degenerate entity. |
| `write.entity.mleader` | `AcDbMLeader` | Requires an `AcDbMLeaderStyle` (resolved from the MLEADERSTYLE dictionary) **and** a content object (an `AcDbMText` content or a content block reference) set via `setContents`/`setEnableAnnotationScale`/`setMText`. The accoreconsole headless seed DB does not carry a usable mleader style + content, and synthesizing a valid MLeaderStyle dictionary entry hostless is a separate (M08-style) ticket. No fake. |
| `write.entity.table` | `AcDbTable` | Requires an `AcDbTableStyle` id (from the TABLESTYLE dictionary) and a row/column/cell layout (`setSize`, `setRowHeight`, `setColumnWidth`, `setTextString` per cell). Not constructible into a *valid* table hostless without a table style present in the seed DB. Deferred honestly. |

These keep returning the honest `OPERATION_NOT_IMPLEMENTED` via the gate (the dispatcher's
final else), and the unit test asserts they never leak into `m08hHasOp`.

## Safety / invariants verified
- **Staged-write only**: handlers append into the staged in-memory model space (`ctx.pDb`,
  `kForWrite`). No `save(` / `saveAs` / `writeDwgFile` / `_QSAVE` anywhere in code — the
  original DWG bytes are untouched. (Asserted source-side over comment-stripped code.)
- **No command stack**: no `acedCommand` / `acedCmd` / `acedInvoke` (compile-disabled in
  2027 and would mutate the attended doc).
- **HasOp <-> Dispatch lockstep**: every admitted op has a handler; every handler is
  admitted; a final `OPERATION_DISPATCH_MISMATCH` guard prevents silent fall-through.
- **Arg validation**: dimension/hatch handlers emit `MISSING_ARG` when required points are
  absent (no fabricated degenerate entities).
- **UTF-8 fidelity**: `njsonStr` for output, `utf8ToWide` for content; the lossy
  `wideToAscii(` funnel is absent.
- **Memory**: each `new`'d entity is either `close()`d after a successful
  `appendAcDbEntity` or `delete`d on failure (centralized in the `m08hAppend` helper; the
  hatch deletes on any pre-append `setPattern`/`appendLoop`/`evaluateHatch` failure).

## Headers / linkage
All entity classes are core AcDb (dimensions in `dbdim.h`, `AcDbText` in `dbents.h`,
`AcDbMText` in `dbmtext.h`, `AcDbHatch` in `dbhatch.h`) exported by the same acdb import
libs that already link `write.entity.line`/`circle`. **No extra `#pragma comment(lib,...)`
was required** (unlike the M08D BRep group, which needed `acbr26.lib`). The `.inc` adds
`#include`s for `dbdim.h` and the AcGe 2D-array headers (`gept2dar.h`, `gedblar.h`,
`gepnt2d.h`, `gevec3d.h`) for the hatch loop; the rest were already included by the TU.
