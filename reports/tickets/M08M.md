## Approach (no fakes)

The OPM property protocol IS the `AcRxProperty` member protocol over an `AcRxClass`
(`AcRxMemberCollection` + `AcRxAttribute`). All of it is **queryable + extendable hostless** (the
classes are exported `ACBASE_PORT` from `acbase`; standard classes, no LNK2001). The Properties
PALETTE / COM `IPropertyManager` surface is attended and is NOT built here.

- **Inspection** ops read the live class member collection (`AcRxClass::members()` →
  `AcRxProperty::cast` per member) and the value protocol (`getValue`/`isReadOnly`/`AcRxValue::toString`).
- **OPM + property-authoring** ops construct the variant-appropriate real `AcRxAttribute` (or
  `AcRxEnumTag`) and attach it to a **transient runtime `AcRxClass`** (`newAcRxClass` →
  `attributes().add()` → `attributes().get(desc)` read-back), then tear it down
  (`deleteAcRxClass`). This is the M08K runtime-class pattern — a real attach to a real registered
  descriptor, not a stub. (Defining a NEW COMPILED property body at runtime is impossible; the
  attribute attach on a real descriptor is the honest hostless proof.)
- **Reactors** subclass the real base classes and **prove the callback fires**: a scratch `AcDbLine`
  is appended, modified (`setEndPoint`), and erased inside an `AriadneStagedWriteTransaction` whose
  dtor **rolls everything back** — the reactor's fire-counter is read before rollback. Every
  `addReactor`/`addPersistentReactor` is balanced by `removeReactor`/`removePersistentReactor`.

## Deferred (11) — attended, OUT of HasOp (honest blocker)

These need the running editor / document manager / long-transaction (workspace) subsystem and
**cannot fire hostless** in `accoreconsole`. Implementing them here would require a live message
loop / attended app, so they are left out of `m08mHasOp` (they keep returning the honest
`OPERATION_NOT_IMPLEMENTED`). No fakes.

| op | blocker |
|----|---------|
| `react.editor.command_monitor` | `AcEditorReactor::commandWillStart/Ended` — fires only from the running command engine (no editor in coreconsole). |
| `react.editor.input_monitor` | `AcEditorReactor` pointer/input notifications — interactive editor input only. |
| `react.editor.lisp_monitor` | `AcEditorReactor::lispWillStart/Ended` — AutoLISP evaluator events; editor-bound. |
| `react.editor.sysvar_monitor` | `AcEditorReactor::sysVarWillChange/Changed` — system-variable change events from the editor. (Note: DB-level `headerSysVar*` exists on `AcDbDatabaseReactor` but the *editor* sysvar monitor is attended.) |
| `react.editor.dwg_lifecycle` | `AcEditorReactor` document open/save/close lifecycle — needs the editor's document events. |
| `react.docmanager.attach` | `AcApDocManagerReactor` — requires `acDocManager` (the document manager), null in pure coreconsole DB jobs. |
| `react.docmanager.monitor` | `AcApDocManagerReactor::documentActivated/...` — attended document-manager events. |
| `react.longtx.attach` | `AcApLongTransactionReactor` — the long-transaction (check-out/check-in between workspaces) subsystem is an editor/workspace concept; its reactor header is not on the hostless DB include/link path. |
| `react.longtx.detach` | as above. |
| `react.longtx.monitor` | as above — `beginCheckOut`/`endCheckIn` fire only from a live long transaction. |

## Invariants honored

- Hostless on `ctx.pDb`; **no** `save()`/`saveAs()`/`_QSAVE`/`writeDwgFile`; **no**
  `acedCommand`/`acedCmd`/`acedInvoke` (asserted source-side).
- Every staged scratch mutation runs in `AriadneStagedWriteTransaction` and is **rolled back** (no
  `commit()` anywhere; asserted).
- Reactor hygiene: `addReactor`/`removeReactor` and `addPersistentReactor`/`removePersistentReactor`
  balanced (asserted); every transient `AcRxClass` torn down with `deleteAcRxClass`.
- All emitted strings via `njsonStr` (UTF-8); no lossy `wideToAscii(` funnel (asserted).
- No Ariadne custom-class `::desc()`/`::cast()` referenced (LNK2001 wall avoided); custom classes,
  where touched, go through `acrxClassDictionary->at(...)`.
