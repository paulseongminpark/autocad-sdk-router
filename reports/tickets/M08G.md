## Hard-safety posture
- **Staged-write ONLY** on `ctx.pDb`. No `save()`/`saveAs()`/`writeDwgFile`/`_QSAVE`; no
  `acedCommand`/`acedCmd`/`acedInvoke`. Original DWG bytes stay unchanged. (Asserted source-level in
  `tests/unit/test_m08g_handlers.py::test_no_original_dwg_write_or_command_stack`, which strips comments
  first so the safety prose that documents the prohibition is not a false positive.)
- Ownership hygiene: every `new`'d entity is `close()`d on append success and `delete`d on append
  failure (centralized in `m08gAppendEntity`). Caller-owned heap arrays returned by
  `explode`/`getOffsetCurves`/`getSplitCurves`/`getSpline` and `AcDbRegion::createFromCurves` are
  appended-or-freed in `m08gAppendPieces` (no leaks, no dangling DB objects).
- All emitted strings route through `njsonStr` (UTF-8 fidelity); the lossy `wideToAscii` funnel is absent.
- On any ObjectARX failure: structured `emitNativeError(r, CODE, msg)` then `return true` — never a fake `"ok"`.

## Deferred ops (20) — honest blockers (left OUT of `m08gHasOp`; keep returning `OPERATION_NOT_IMPLEMENTED`)

### ASM modeler required — NOT linkable hostless (10)
`AcDb3dSolid::createBox/createExtrudedSolid/createLoftedSolid/createRevolvedSolid/createSweptSolid`,
`AcDbSurface`/`AcDbNurbSurface`/`AcDbBody`/`AcDbSubDMesh` creation, and `AcDb3dSolid` boolean
(`booleanOper`) all dispatch into the ShapeManager/ASM modeler kernel, which is not available/linkable
in the headless `accoreconsole` (ObjectDBX) host. Attempting them would require the modeler runtime; per
the no-fake-success rule they are deferred rather than stubbed:
- `write.entity.solid3d.primitive`, `write.entity.solid3d.extrude`, `write.entity.solid3d.loft`,
  `write.entity.solid3d.revolve`, `write.entity.solid3d.sweep`, `modify.entity.solid3d.boolean`
- `write.entity.body`, `write.entity.surface`, `write.entity.nurbsurface`, `write.entity.subdmesh`

### External resource / dependency object required (6)
- `write.entity.rasterimage` — needs an `AcDbRasterImageDef` + the raster (ISM) engine; no image def in a headless staged db.
- `write.entity.wipeout` — `AcDbWipeout` is an `AcDbRasterImage` subclass requiring the same raster/image-def machinery + the WipeOut RX app.
- `write.entity.shape` — `AcDbShape` references a loaded SHX shape file (shape index/name); no SHX is registered in the staged db.
- `write.entity.mline` — `AcDbMline` requires a named `AcDbMlineStyle` in the MLINESTYLE dictionary; standalone creation without a style is invalid.
- `write.entity.mpolygon` — `AcDbMPolygon` lives in the AcMPolygon module and depends on the hatch/area (gradient/boundary) engine; not a plain DB ctor.
- `write.entity.polyfacemesh` — `AcDbPolyFaceMesh` needs post-residency vertex append + `AcDbFaceRecord` index wiring (faces reference 1-based vertex indices); deferred to avoid emitting a half-wired, invalid mesh (no fake partial).

### Owning-instance / editor-bound subentity association (4)
- `write.entity.attribute` — `AcDbAttribute` is only valid appended to an owning `AcDbBlockReference` instance (not standalone in model space); the create-standalone contract does not apply.
- `edit.subentity.add_paths`, `edit.subentity.delete_paths`, `edit.subentity.transform` — operate on
  `AcDbFullSubentPath`/`AcDbAssoc*` subentity paths, which require subentity picks/marker context and the
  associative framework; deep + editor-bound, deferred per the brief's guidance.
