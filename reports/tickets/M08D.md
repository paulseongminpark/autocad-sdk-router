as the AcBr import lib. This is **WRONG**, proven by `dumpbin /LINKERMEMBER`:
- `AcDrawBridge.lib` (in `lib-x64`, 704 symbols) = a DrawBridge/Config localization lib, **0 AcBr symbols**.
- The real AcBr import lib is **`C:\ObjectARX 2027\utils\brep\lib-x64\acbr26.lib`** (274 AcBr symbols) +
  `acgex26.lib` (AcGe extensions) — a dir NOT on the project's `AdditionalLibraryDirectories`.
- `AcDb3dSolid` + `AcGeNurbSurface` resolve from **`acgeoment.lib`** (already in the on-path `lib-x64`;
  this is exactly what the SDK's own `brsample.vcxproj` links).

Reached WITHOUT editing the forbidden `.vcxproj`: absolute `#include` of the brep headers +
`#pragma comment(lib, "<abs>\\acbr26.lib")` (+ acgex26.lib abs, + acgeoment.lib by name) inside the `.inc`.
Recommend updating the slice + design §3 lib list in a future doc-only ticket.

## DESIGN NOTES (real reads, no fabrication)
- Each op binds by hex `handle` (`resolveHandle` → `acdbOpenObject kForRead`). Geometry-kernel ops read
  point/scalar args from JSON (`MISSING_ARG` if absent), build the AcGe object, compute, emit — host-less.
- AcBr ops bind `AcBrBrep::set(*pEnt)` under an `AriadneReadTransaction`; a failed bind returns
  structured `BREP_BIND_FAILED` with the AcBr status code (not a fake success).
- Traversal ops return real element COUNTS via the canonical `setBrep/get/next/done` idiom; per-element
  inspect ops bind the FIRST element of that kind and report its property + `index:0` + status code
  (deeper per-index addressing is a documented follow-up — flagged honestly, not faked).
