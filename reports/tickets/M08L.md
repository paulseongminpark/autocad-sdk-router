## Implemented (25)

### T01 — worldDraw / graphics capture (7)

A complete hostless `AcGiWorldDraw` collector drives an entity's view-independent graphics
elaboration with no GS/viewport and tallies the primitives it emits. Collector classes (plain C++
subclasses, no `ACRX_DECLARE_MEMBERS` — nothing queries their RTTI; `worldDraw()` invokes the
virtuals via vtable):
- `M08LCollectWorldDraw : public AcGiWorldDraw` (8 `AcGiCommonDraw` pure virtuals + `geometry()`)
- `M08LCollectGeometry : public AcGiWorldGeometry` (~30 `AcGiGeometry` pure virtuals + `setExtents`)
- `M08LCollectTraits : public AcGiSubEntityTraits` (13 pure virtuals)
- `M08LCollectContext : public AcGiContext` (3 pure virtuals)

| op | mechanism (AcGi class / method) |
|----|----|
| `render.draw.worldgeom` | `AcDbEntity::worldDraw(&collector)` (drawable.h:162, `ACDBCORE2D_PORT`) → tally circle/arc/polyline/polygon/mesh/shell/text/… + first vertex + extents |
| `render.entity.worlddraw_override` | same elaboration, framed as the entity's `subWorldDraw` output (its overridden draw) |
| `render.polyline.helper` | same elaboration; surfaces the `AcGiGeometry::polyline`/`polyPolyline` call count |
| `render.facedata.attach` | same elaboration; surfaces shells/meshes that carried a non-null `AcGiFaceData` |
| `render.traits.set` | `AcGiSubEntityTraits` the entity SET during elaboration (color/lineweight/fill/ltscale/thickness) |
| `render.drawable.def` | `AcGiDrawable` identity: `isPersistent()` / `id()` / `drawableType()` |
| `render.context.query` | `AcGiCommonDraw`/`AcGiContext` surface: `regenType` / `numberOfIsolines` / `isDragging` / `context()->database()` |

Optional `regen_type` arg maps to `kAcGiStandardDisplay` (default) / `kAcGiShadedDisplay` /
`kAcGiForExplode` / `kAcGiSaveWorldDrawForProxy` / `kAcGiHideOrShadeCommand` (acgidefs.h).

## Deferred (2) — honest, OUT of `m08lHasOp`

| op | concrete blocker |
|----|----|
| `render.draw.viewportgeom` | Requires an `AcGiViewportDraw` (acgi.h:870) bound to a live **viewport** — `AcGiViewportDraw::viewport()` returns `AcGiViewport&`, and the entity's `viewportDraw()` path needs a real GS viewport context that does not exist headless. `worldDraw` (view-independent) is the only hostless graphics elaboration surface. No fake: a stub `AcGiViewportDraw` could be constructed but the entity's viewport-dependent geometry would have nothing meaningful to elaborate against. |
| `overrule.dimstyle.install` | **No `AcDbDimStyleOverrule` class exists in ObjectARX 2027.** Header scan of `inc\*overrule*.h` (rxoverrule.h, dbobjectoverrule.h, dbentityoverrule.h, drawable.h) yields Object/Properties/Osnap/Transform/Grip/Subentity/Highlight/HighlightState/Visibility/Geometry/Drawable/QueryX overrules only — no dimstyle base to construct + `addOverrule`. Registering against a non-existent class is impossible; faking it would be dishonest. |
