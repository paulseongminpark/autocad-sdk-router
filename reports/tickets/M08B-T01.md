## Runtime smoke — finding + deferral (honest, not a skip)

A live smoke was attempted (worktree router + the now-tracked `tests/fixtures/native_sample.dwg`). **Finding
(architecture):** the router selects engine **by op**, and routes catalogued ops — including native-family ones
(`compute.brep.massprops`, `extend.customobject.define`) — to the **managed** `CadJobRunner`
(`engine=managed_objectarx_active_document`), which returns its own *"Unsupported CAD job operation"*. It does
**not** route them to the native `.crx`. So this ticket's native `OPERATION_NOT_IMPLEMENTED` is a **native-module-
internal** contract that fires only on a **direct `ARIADNE_NATIVE_JOB`** invocation (the router does that only for
specific ops like `inspect.database.graph` + P1 writes). The router's per-op smoke therefore cannot reach the
native gate.

- **Deferred:** the direct-`ARIADNE_NATIVE_JOB` negative-path smoke (load worktree `.crx` in accoreconsole, feed a
  catalogued op, assert `error_code=OPERATION_NOT_IMPLEMENTED` + `engine=native_objectarx`). This is an
  attended/direct-native harness item for **MERGE-M08-READ** (as M07 ran attended verification separately). The
  former blocker — "worktree has no DWG" — is now **removed**: `tests/fixtures/native_sample.dwg` (AC1032) is
  tracked, so any worktree can stage it.
- **Why this is safe to defer:** the native gate is **compile-verified** (build exit 0; the `.crx` grew by the
  added code) and **source-contract-verified** (table↔handler parity + the gate present in `ariadneNativeJob()`).
  The gate is a deterministic 3-line lookup→emitNativeError→return now proven present in the built binary; and the
  39 existing handler bodies are byte-identical (259 tests pass) so the native happy path cannot regress.
